# 0.59.0 — 2026-06-25

Organising your pages just got hands-on. You can now **drag a page to re-nest
it**, **filter the list by tag from a searchable dropdown**, and **set a page's
icon** yourself — the icon was previously only changeable by the assistant.

## Highlights

- **Drag-to-reparent in the pages tree.** Every row in `/pages` now has a grip
  handle (it lives in a fixed left gutter, so handles line up no matter how deep
  a page is nested). Drag a page onto another to nest it underneath, or onto the
  "move to the top level" band that appears at the top while dragging to pull it
  back out. The whole subtree travels with it, and a drop that would create a
  cycle (onto itself or one of its own sub-pages) is refused. There's also a
  **"Move to…" menu** on each row — a keyboard- and precision-friendly path that
  lists every valid destination (self + descendants excluded). This is the
  user-facing half of the page-move capability the assistant gained in 0.58.0;
  both go through the same `movePage` (parent + ltree path rewrite, cycle guard).

- **Searchable tag filter.** The tag row used to lay every tag out as a pill,
  which got unwieldy past a handful. It's now a single compact dropdown beside
  the sort control: open it, **type to search tags**, and pick one (each row
  shows its page count). "All pages" or the **Clear** button resets the filter.
  Same URL-driven SSR filtering underneath — just far tidier with 20+ tags.

- **Set a page's icon.** Click the icon above the title in the editor to open a
  lightweight emoji picker — a curated, searchable set grouped by theme (docs,
  ideas, people, objects, nature), plus **Remove icon** to fall back to the
  default. The choice saves with the page's title/tags metadata and shows up
  everywhere the page appears (tree, list, preview, drag ghost, sub-page cards).

## Notes

- The emoji picker is dependency-free — a curated set rather than the full
  Unicode catalogue, to keep the web bundle lean. Common document icons are
  covered; the assistant can still set any emoji via `page_update`.
- A cleared icon is stored as a blank and normalised back to "none" on read
  (`rowOf`), so removing an icon restores the default glyph consistently across
  every surface.
- No schema migration in this release — icons already lived in `nodes.data.icon`
  and the page move/PATCH routes were already in place.
