# v0.138.0 — Share a page, share the whole section

Sharing a page can now take its sub-pages with it — one switch to publish an
entire documentation tree, kept in lockstep with the parent.

## Features

- **Share sub-pages.** A page's Share popover gains a **Share sub-pages**
  toggle whenever the page has children. Turn it on and every sub-page is
  shared too; turn it off — or un-share the parent — and the child links go
  with it. The children mirror the parent: flip a page between public and
  **team-members-only** and its whole shared subtree follows. Snapshot
  semantics — a sub-page you add *later* isn't auto-shared, so re-toggle to
  pick it up. Built for a linked index (SOPs, QMPs, registers) whose documents
  should open for the same audience as the index itself.

- **The Team Hub shows one card per shared tree.** A team-shared page nested
  under another team-shared page no longer gets its own hub card — the top of
  the tree stands in for the whole thing, so cascade-sharing a big index
  doesn't flood the hub. The nested pages stay openable from links inside the
  parent.

- **The assistant can share a whole section from chat.** `page_share` now
  takes `mode` (public / team) and `children` (share the subtree), and
  `page_unshare` removes a shared subtree in one call — so *"share the
  documentation index and all its sub-pages with the team"* just works. The
  Tables (Ledger) specialist can now share what it produces, alongside the
  persona and the Pages specialist.

## Under the hood

- Cascade intent lives in `shares.settings.cascade` on the parent share — no
  migration; children are ordinary shares. Descendants come from the
  `parent_id` recursion (`listPageDescendantIds`), and mode changes / un-share
  propagate through the cascade-aware `applyShareMode` and `revokeShareTree`.
- The hub's top-most-only filter is an ltree `<@` `NOT EXISTS` in
  `listTeamHubSections` — a page whose ancestor page is also team-shared is
  left off the card list.
- Owner API: `POST /api/shares/cascade { nodeId, on }`; `GET /api/shares`
  now returns `childCount` so the switch appears only when there are sub-pages.
