# 0.102.0 — 2026-06-30

**A fresh install that just works.** This release closes a set of gaps an audit
found on the path from clone to a running brain — the kind of "looks set up but
quietly broken" problems that only bite after deploy. A new box is now correct by
construction, not just diagnosable after the fact.

## Highlights

- **The object-store bucket is created automatically on a fresh deploy.** The
  production stack now provisions the MinIO bucket as a one-shot step before the
  app starts. Previously a freshly-pulled box came up with an empty store and the
  first upload, app build, or file sync failed with "The specified bucket does not
  exist" until someone created it by hand. Every service that writes files waits
  for the bucket to exist.

- **Mini-app data survives updates.** `/apps` SQLite databases are now on a
  persistent host volume instead of ephemeral container storage, so an app's data
  is no longer lost when you update the stack.

- **Onboarding can't finish onto a dead brain.** The wizard now refuses to
  complete until your assistant is actually provisioned (an API key was added and
  setup ran), and the onboarding gate is enforced on the server — so you can't
  land in an empty, non-answering app. If something's missing, it tells you what.

- **In-app updates work on manual installs.** The example production env now
  documents `MANTLE_STACK_DIR` (the one setting the in-app updater needs), so a
  hand-configured deploy no longer ends up with Settings → Updates hanging. A few
  other optional knobs (CORS origins, trusted-proxy depth, push relay, Microsoft
  365) are surfaced there too.

- **Install docs rewritten.** The getting-started guide now matches reality — the
  one-line installer and the in-app signup + onboarding wizard — instead of the
  old hand-edit-the-database steps. Self-hosting, deploy, and onboarding docs were
  brought up to date alongside it.
