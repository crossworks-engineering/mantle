# v0.129.1 — Browser sidecar on the dashboard

The PDF-export browser sidecar (v0.129.0) now reports on the **System
vitals** card, with a **Browser** pill beside Tika's:

- **Green** — sidecar healthy; hover shows the browserless + Chromium
  versions (probed from the same `/meta` endpoint the compose healthcheck
  uses).
- **Red** — configured but unreachable: PDF downloads will 503 until it's
  back (Markdown/Word unaffected).
- **Neutral** — `BROWSER_WS_ENDPOINT` not set (e.g. detached frontend dev),
  where a red dot would be noise.
