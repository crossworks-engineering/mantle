# 0.160.1 — 2026-07-22

A patch release. One fatal boot bug fixed, plus repository hygiene.

## Fixed

- **The API died at boot on every box.** pg-boss 12 changed `updateQueue` to
  reject the `policy` key *unconditionally* — it throws whenever the key is
  present, even when the value matches what the queue already has:

  ```js
  if ('policy' in options) throw new Error('queue policy cannot be changed after creation');
  ```

  `startExtractQueue` passed `queueOptions` through wholesale, so the extract
  queue threw during startup and took the API process with it. The fix strips
  the key before the update ([apps/api/src/agent/extract-queue.ts](../../apps/api/src/agent/extract-queue.ts)):

  ```ts
  const { policy: _policy, ...mutableQueueOptions } = queueOptions;
  await boss.updateQueue(EXTRACT_QUEUE, mutableQueueOptions);
  ```

  `createQueue` still receives the full options, so the policy is set correctly
  at creation and only the *update* path is narrowed. This was the sole call
  site in the tree.

  This bug arrived with the pg-boss 10 → 12 jump in
  [v0.159.0](./0.159.0.md) and made that release undeployable on its own.

## Changed

- **`apps/agent/` is now gitignored.** It is a local scratch area, not part of
  the build, and a blanket `git add -A` had swept a personal file into a commit.
  The file was removed from the tree in v0.160.0 and its history purged
  afterwards; the ignore rule prevents a recurrence.

## Housekeeping

`main` was failing `pnpm verify` before this release — both failures predate it
and neither was caught by the checks run at the time (typecheck, tests, and a
production build all passed; `verify` chains typecheck → **lint** → **format** →
tests, and the last two links were never exercised):

- **2 lint errors** — `unitDoc` / `unitRow` in
  [packages/content/src/table-formula-diff.test.ts](../../packages/content/src/table-formula-diff.test.ts)
  were assigned and never read, left over from an earlier draft of that test
  block. Every test in it calls `raw()`, which evaluates mathjs directly and
  needs neither. Deleted.
- **12 files failing `prettier --check`** — all of them v0.160.0 formulas work.
  Reformatted; the change is cosmetic (32 insertions, 36 deletions, no logic).

`pnpm verify` is green as of this release: 230 test files, 2688 tests passing.

## Verified

The pg-boss 12 path was exercised end to end against the dev brain after this
fix — API boots clean with no fatal, `mantle.extract` reports `policy=short`,
70+ jobs completed, and crash recovery behaves: a worker killed mid-extraction
has its lease expired at the 60-minute budget and requeued **without** charging
the 5-retry budget (`retry_count` stayed 0). The `already_extracted` guard in
[apps/api/src/agent/extractor.ts](../../apps/api/src/agent/extractor.ts) then
short-circuits the replay, so a crash-and-retry costs no LLM spend and does not
overwrite an existing summary.

## Upgrading

No schema migration in this release. **Coming from v0.158.x, the pg-boss 10 → 12
schema step from [v0.159.0](./0.159.0.md) still applies** — the `pgboss` schema
must be dropped and rebuilt per box, and rolling the image back does not roll
the schema back. Runbook: [docs/pg-boss-12-upgrade.md](../pg-boss-12-upgrade.md).
