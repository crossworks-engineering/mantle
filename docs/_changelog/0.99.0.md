# 0.99.0 — 2026-06-30

**Two new ways to bring your own model.** Point a chat route at **GitHub
Copilot** (use your Copilot subscription's frontier models) or at **any
OpenAI-compatible cloud endpoint** — Z.ai/GLM, Qwen/DashScope, Moonshot/Kimi,
Nvidia NIM, Together, Fireworks, DeepInfra, Azure AI Foundry, and the like —
without waiting for a bespoke integration. Plus groundwork for streaming the
model's *real* reasoning instead of canned "thinking" phrases.

## Highlights

- **GitHub Copilot provider.** A new chat provider backed by your Copilot
  subscription — GPT, Claude, Gemini, and o-series models behind one key. The
  "API key" is your GitHub Copilot OAuth token; Mantle exchanges it for the
  short-lived Copilot token automatically and refreshes it as needed.
  Reasoning-capable models are driven via the effort control.

- **Custom (OpenAI-compatible) provider.** One provider for the whole long tail
  of cloud endpoints that speak the standard OpenAI `/chat/completions` API. Pick
  **Custom**, set a Base URL and API key on the route, then pick or type the
  model id — that's it. (For a model on your own hardware, keep using
  **Local**; Custom is the keyed-cloud counterpart, no Tailscale plumbing.)

- **Cleaner output from open models.** Models that emit their chain-of-thought
  inline as `<think>…</think>` (DeepSeek-R1, Qwen QwQ, many local GGUF builds) no
  longer leak that reasoning into the reply or the saved message — it's stripped
  as it streams, across every OpenAI-compatible route.

- **Real reasoning, opt-in.** The plumbing to request, stream, and surface the
  model's actual reasoning (a collapsible "Thinking" trace) — replacing the old
  rotating canned phrases — is in place behind a per-box flag
  (`MANTLE_THINKING_BUDGET`). Off by default while it's proven against live
  provider calls; the tool-step narrator continues to label each action.
