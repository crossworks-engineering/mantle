# 0.47.0 — 2026-06-24

SharePoint and OneDrive files now delta-sync into the brain as ordinary,
searchable file nodes.

## Highlights

- **Microsoft M1 — SharePoint/OneDrive delta sync.** Enabled drives delta-sync
  into Mantle, reusing the email-attachment ingestion path end to end:
  `@mantle/microsoft/drives` discovers OneDrive + followed SharePoint libraries,
  syncs the Graph `/root/delta` feed (download → store; tombstones → remove),
  and stores each file sha256-deduped — exactly like an email attachment.
- **Synced files are just `type:'file'` nodes.** The key reuse decision: no new
  node type, so the existing extractor parses / summarizes / embeds them
  unchanged and they appear in search and the knowledge graph like anything
  else.

## Notes

- `ms_drives` (opt-in, per-drive delta cursor) + `ms_drive_items`
  (provenance / dedup) schema + **migration 0102**. A `microsoft-sync` pg-boss
  worker runs every 2 min (single-flight per drive) via the `worker_microsoft`
  compose service.
- `/settings/microsoft` gains a drive picker: "Refresh drives" + per-drive
  opt-in toggles.
- v1 is flat-layout, size-bounded, followed-sites-only. Migrations 0100–0102
  still pending apply (prod DB).
