# 0.146.1 — 2026-07-18

**Deleting an agent no longer fails when it has chat history.** The
`assistant_messages.agent_id` column carried an ON DELETE SET NULL foreign
key (a deleted agent orphans-but-preserves its archive) but had been made
NOT NULL — contradictory, so deleting any agent that ever exchanged a web
chat message hit a constraint violation and a 500. The column is nullable
again: the delete succeeds and the agent's conversation history is
preserved for recall and digests, unattributed. Deleting history remains a
deliberate act, never a side effect.
