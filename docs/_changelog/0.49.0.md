# 0.49.0 — 2026-06-24

The responder now reaches for far more of a long document before it answers, so big procedures and standards get real context instead of a keyhole view.

## Highlights

- **More passage context per turn.** The default `chunk_limit` rose from 3 to
  12. A 64-page procedure chunks into 100+ passages; at 3 the responder was
  answering from ~2% of the document. 12 (~18k chars / ~4.5k tokens) gives
  modern long-context models enough of the document to reason over without
  forcing a full file read every turn. A per-agent `memory_config.chunk_limit`
  still overrides.

- **Read the whole document when it matters.** The `tool_grounding` skill now
  tells the responder to open the full document (`file_read` / `node_read`)
  when an answer turns on a procedure, standard, checklist, spec, or table —
  anything where steps, conditions, or rows must be read in full and in order —
  rather than stitching an answer from scattered chunks.

## Notes

- The `chunk_limit` default lives in the runtime (`CHUNK_LIMIT_DEFAULT`), so it
  reaches existing brains on image update — responders carry no explicit
  override. The manifest persona mirrors `chunk_limit: 12` for fresh onboarding.
- No schema migration in this release.
