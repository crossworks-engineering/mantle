# v0.153.3 — Stop-fix audit hardening

Fixes from two adversarial review passes over v0.153.2's "Stop halts
tool-heavy turns" change. No high-severity findings; the loop checks
themselves held up (pairing invariant, no strandable pending rows, zero
change for signal-less contexts like Telegram/heartbeats).

## What changed

- **No more red "N of M tool calls failed" on a stopped turn** — the new
  `cancelled_by_user` skip reason wasn't in the loop's `SKIP_REASONS` set, so
  cancelled calls counted as _failures_ in the persisted tool-outcome ledger
  and lit the always-visible destructive badge on a turn the user
  deliberately ended. They now count as skipped; the hover tooltip reads
  "not run (guards or Stop)".
- **Stopping never blanks the reply you were reading** — a Stop landing in a
  round whose own text was empty (models usually emit tool calls without
  prose) finalized the durable row with `''`, and reconcile replaced the
  visible streamed commentary with an empty bubble. The loop now carries the
  last non-empty round's text into the stopped result.
- **No phantom activity after Stop** — cancelled calls no longer open trace
  steps (each one was publishing a fresh "doing X" thought-trail line _after_
  the user pressed Stop) and are excluded from the persisted thought trail
  (they never ran).
- **A Stop on the non-streaming path settles immediately** — with
  `MANTLE_TURN_TOKENS` off, a Stop surfaces as a thrown AbortError from the
  one-shot `chat()` call. The retry wrapper treated that as transient
  (backoff sleeps against a dead signal) and the tool loop then _failed over
  to the backup route_ with false "primary failed" telemetry. `withChatRetry`
  now short-circuits when the caller's signal is aborted, and the loop
  rethrows on `turnAborted()` before the failover check.
- Post-batch stop no longer duplicates the round's text as a second
  assistant message in the returned transcript (inert today, wrong for any
  future consumer).

Covered by 5 new tests (9 total in `tool-loop.abort.test.ts`: cancelled ⇒
skipped-not-failed, last-round-text fallback, no-failover-on-abort, no
pending-confirm row post-abort) plus 2 in `retry.test.ts` (abort ⇒ no retry;
bare timeout AbortError still retries). 195 tests pass across
agent-runtime / voice / assistant-runtime.
