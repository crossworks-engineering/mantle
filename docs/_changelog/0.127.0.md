# v0.127.0 — Assistant chat window rework

The global assistant grew into a real workspace companion: it keeps its
session, one responder stays in charge everywhere, and the panel can shrink
into a docked column so the work stays visible while you chat.

## Three display modes, your pick remembered

- **Full display** — the assistant fills the content area (header, nav, and
  activity rail stay framed around it), with the prompt-in-the-margin turn
  layout.
- **Docked column** — the panel shrinks to a right-hand column between the
  content and the activity rail (lg+ viewports); the screen you're working on
  reflows beside it and stays fully interactive. The transcript layout is now
  container-query driven, so it stacks gracefully at column width.
- **Minimised** — the familiar bubble.

The header toggle flips full ⇄ docked on any screen; the preference persists
across screens and sessions. Docked is the default — it's the mode that keeps
your work visible.

## The chat keeps its session

The thread no longer remounts as you move around the app, and the composer
draft survives navigation, minimise/restore, agent switches, and even a page
reload (persisted per agent for the browser session). Type, wander off, come
back — everything is where you left it.

## One responder, everywhere

Opening a page, table, or app no longer switches the chat to that surface's
specialist. Your selected responder stays in charge on every screen and
delegates to the Pages/Tables/Apps specialists herself (`invoke_agent`) — she
has the fuller context and plans the work. The open node is pinned as context
on every turn, phrased so a delegating responder hands the node id (and any
focus directive) to the specialist verbatim.

## The marker you can trust

Marking sections for the assistant on a page is now unambiguous:

- Every marked block shows as a **chip in the composer with a snippet of the
  block's text** — remove one with its ×, or clear the set.
- A context strip under the chat header reads *Working on ‹page› ·
  N sections focused · N draft changes to review*.
- In the docked column, the editor sits right beside the chat: gutter marks,
  live edits, and the review diff are visible as they happen — the draft →
  review → Commit flow is unchanged.
- Transcript prompt cards show what you typed plus a quiet "Sent with
  on-screen context" footer (hover for the full appended context), instead of
  the raw machine preamble.

## Fixes

- **CORS**: the `Idempotency-Key` header is now allowed in the API preflight —
  assistant/team turn sends from detached clients (Electron, `pnpm dev:fe`)
  previously never left the browser. Same-origin installs were unaffected.
- The page editor's toolbar wraps instead of clipping when the pane narrows
  (docked column, small windows) — Mark/Revert/Share stay reachable.
- One focused-section chip per mark id on legacy docs that still carry
  pre-v0.120.1 duplicate block ids.
