# 0.37.0 — 2026-06-23

The manifest single-source-of-truth pattern is now documented — and the last
propagation gap is closed.

## Highlights

- **A change-and-propagation rulebook for the manifest.** New
  `apps/web/lib/system-manifest/CLAUDE.md` spells out the one rule —
  default agents, skills, tool groups, workers, and the persona's structure all
  live in `manifest.ts`; never hardcode a model/prompt/grant/worker anywhere else
  — plus a table of exactly what reaches an existing brain on upgrade vs. what
  stays operator-owned. The parent `apps/web/CLAUDE.md` and the `manifest.ts`
  header now point at it, so any session editing config finds the contract first.

- **Specialist skill links now propagate too.** The boot reconcile already
  unioned a specialist's manifest tool groups onto existing brains; it now does
  the same for skill links. So a new skill wired to an existing specialist (not
  just the persona) reaches an already-provisioned brain on the next version
  bump — closing the last surface that only reached fresh installs. Additive and
  enabled-only, like every other reconcile grant.

## Why this matters

Across recent releases every config surface the sanity system tracks became
manifest-driven and upgrade-safe. This release writes that down so future changes
follow the pattern by default, and removes the one remaining asterisk (specialist
skills) so the documented contract is complete and true.

## Notes

- No schema migration in this release.
