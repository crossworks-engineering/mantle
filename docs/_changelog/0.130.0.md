# v0.130.0 — Footer status bar

Navigation and the assistant controls move into one **full-width footer bar**,
plus two smaller polish fixes to Pages and mini-apps.

## One control strip

The floating Assistant/Mark bubbles are gone. In their place is a footer that
runs the full width of the content area and gathers every shell control in one
predictable spot:

- **Start** — the sidebar collapse toggle (⌘B), sitting under the sidebar it
  controls.
- **Centre** — your **five most-used menus**, ranked from actual usage
  (tracked per-browser in localStorage; never sparse — it falls back to sidebar
  order until you've clicked around). Labels on wide screens, icons when tight.
- **End** — the **Highlight content** and **Assistant** launchers as labelled
  buttons (no more hunting for a round bubble), the assistant's **full-display ⇄
  side-column** toggle (shown only while the assistant is open — it used to hide
  in the assistant window's chrome), and the Activity collapse toggle (⌘J).

The two collapse toggles now live in the footer instead of on the rails they
control, mirrored so they read as a matched pair. Every full-height region
(main, sidebar, Activity rail, assistant panel, mail, fleet) ends above the bar
via the new `--footer-h` shell variable, so nothing hides behind it.

## Pages: solid H1

The Pages/Notes H1 was painted with a `primary → chart-3` gradient via
`background-clip: text`. The transparent glyph fill it needs was fragile — it
fought PDF export and in-editor caret handling — so headings are now a solid
brand colour. Same theme token, none of the breakage.

## Mini-apps: themed scrollbars

Sandboxed apps rendered the browser's default wide scrollbar with a white/grey
track that clashed with the active theme. The app iframe now carries thin,
theme-token scrollbars (scoped to the sandbox), so an app's scroll areas look
like the rest of Mantle in every theme × light/dark.
