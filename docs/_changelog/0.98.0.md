# 0.98.0 — 2026-06-29

**Drop a spreadsheet in, get a Table out — automatically.** Uploading an
`.xlsx` / `.xls` / `.csv` now builds typed Table(s) on ingest, however the file
arrived (Files screen, chat attachment, email, Telegram). Registers stop being
search-only text blobs and land in `/tables` as real, queryable grids.

## Highlights

- **Auto-import on ingest.** The extractor turns a spreadsheet file into typed
  Table(s) — one per non-empty sheet — published and indexed immediately, at the
  shared `node_ingested` chokepoint so every upload path is covered. Deduped by
  the source file (`data.sourceFileId`), so re-processing the same file never
  creates a second table.

- **Big sheets paginate, they don't truncate.** A sheet over `MAX_GRID_ROWS`
  (10,000; env `MANTLE_MAX_GRID_ROWS`) is split into contiguous parts — "… (part
  1/N)", same columns — so no rows are lost while no single table balloons its
  JSONB grid. The auto-importer also caps total tables per upload
  (`MAX_AUTO_TABLE_TABLES`, 20) so a many-sheet workbook can't fan out into a
  burst of indexing passes. The shared parser means the manual `table_from_file`
  tool and the UI import paginate the same way.

- **The assistant knows a spreadsheet from a document.** When you attach a
  spreadsheet in chat, the assistant is told it's been imported into Tables (read
  it to discuss the data) instead of being nudged to import it as a page. It can
  also import one on request directly via a focused `tables-import` capability —
  full grid authoring still lives with the Ledger specialist.

- **Chat overlay polish.** The global chat overlay now minimises on any
  navigation (not just the nav rail) and keeps itself pinned to the latest
  message as content settles.
