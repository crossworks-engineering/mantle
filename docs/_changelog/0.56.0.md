# 0.56.0 — 2026-06-24

Every item is now linkable. Responders can drop a clickable link to a note,
page, todo, table, event or file in a reply, and the user taps straight through
to it.

## Highlights

- **Universal permalink — `/n/<id>`.** One canonical, type-agnostic deep link
  to any node. The route loads the node owner-scoped, reads its `type`, and
  redirects to the surface that edits/displays it — note → `/notes?selected=`,
  page → `/pages/<id>`, todo → `/todos?selected=`, table → `/tables?selected=`,
  app/event → `/apps|events/<id>`, file → `/files?file=`, contact →
  `/contacts?id=`, life-log → `/lifelog?selected=` — falling back to the
  universal `/nodes/<id>/history` biography for any type without a dedicated
  editor. Callers only ever hold an id; the type→surface map lives in this one
  route, so links survive a surface changing its URL shape.

- **Responders link items themselves.** `search_nodes`, `node_read`, and the
  `note_get` / `page_get` / `todo_get` / `table_get` / `event_get` tools now
  return a `url` field, and their descriptions tell the model to surface items
  as markdown `[title](url)`. The instruction rides the tool descriptions, so it
  reaches every agent that holds the tool with no manifest reseed. `nodeUrl(id)`
  in `@mantle/content` builds the absolute permalink.

- **Links actually navigate in chat.** Replies render read-only through the
  shared page-editor schema, which sets `link.openOnClick:false` (right for the
  editable canvas) — so links rendered but did nothing. The reply renderer now
  intercepts clicks: same-origin links (incl. `/n/<id>`) go through the SPA
  router, external links open in a new tab. The editable Pages canvas is
  untouched.

## Notes

- Deep-linking a todo outside the current page slice now works — `/todos`
  reads `?selected=` and fetches that todo directly so the detail pane opens it.
- Security: the permalink loader is owner-scoped; a leaked id for another owner
  404s rather than leaking existence via a permission error.
- See [`permalinks.md`](../permalinks.md) for the full design.
- No schema migration in this release.
