# 0.51.0 ‚Äî 2026-06-24

Tables gained a real structured lookup: the agent can now find the rows that match a value instead of paging the whole grid.

## Highlights

- **`table_query` ‚Äî query rows by value.** A new read-only tool: give it
  `filters` (`{ column, op, value }`, ops `eq|neq|contains|gt|lt|gte|lte|empty|notEmpty`,
  AND-ed by default, `match: "any"` to OR), optional `sort` and `columns`, and it
  returns only the matching rows (cells by column name, formulas resolved) plus a
  total match count. So "what's the design pressure for circuit X" or "which rows
  are below threshold" are answered directly, even on a 10k-row table.

## Notes

- Built on a pure `queryRows` model function that reuses the exact filter +
  sort logic saved views already use, so a query and a saved view agree.
- Read-only ‚Äî it persists nothing (unlike `table_set_view`). It joins the
  `tables` tool group, so the Tables specialist (Ledger) gains it on existing
  brains via tool-group sync on image update.
- No schema migration in this release.
