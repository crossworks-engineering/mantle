# 0.78.0 — 2026-06-27

**Watch your assistant think.** The `/assistant` chat no longer freezes on a
spinner while a turn runs. You now see a live **thought trail** — what it's
searching, reading, and delegating, narrated in its own voice — and then the
reply **types out token by token** as the model writes it. You can **Stop**
a turn mid-sentence, the turn runs in a durable background process that
**survives navigating away or reloading**, and a dropped connection **resumes
where it left off**. The same live stream feeds the mobile companion.

## Highlights

- **Live status, grounded in what's really happening.** As a turn runs, a thought
  trail shows each step as it opens and closes — "Searching your brain for *SLA*",
  "Reading note *Q3 plan*", "Delegating to *Researcher*" — with a name, an icon,
  and a timer per step. The labels are built from the agent's **actual** tool
  calls and arguments, not a guess, so the trail never confabulates. Delegated
  sub-agents stream into the **same** turn, so a hand-off to Researcher or Pages
  shows its work inline.

- **The reply types out live.** Token streaming runs end to end: the visible
  answer renders as the model produces it, including reasoning where a model
  exposes it. It works across **every chat provider** — OpenRouter (the common
  path, incl. `anthropic/claude-*`), Anthropic-native, and any provider that
  won't stream degrades cleanly to a single render. Tool-call arguments that
  arrive as streamed fragments are reassembled correctly, and token/cost
  accounting still lands on the final chunk.

- **A narrator that restyles status in the assistant's voice.** A new, configurable
  **narrator** AI-worker turns the raw "tool: search_chunks" event into a natural
  one-liner. It's fed **only the real action** ("describe this in ≤6 words, present
  tense, never predict next steps"), runs fire-and-forget off the turn's critical
  path, and its system prompt is the verbosity dial. It auto-seeds on new brains
  and reaches existing ones on upgrade; brains without one fall back to a built-in
  concise label, so there's zero regression.

- **Stop a turn mid-flight.** While a turn runs, the send button becomes a **Stop**
  button. Press it and generation actually halts on **any provider**, the partial
  reply you already received is kept, and **your prompt drops back into the
  composer** so you can tweak and resend. No more waiting out a turn that's clearly
  going the wrong way.

- **Turns survive navigation, reload, and backgrounding.** The turn route now
  returns immediately (**202**) and the client drives entirely off the live
  stream — the work runs in the durable `apps/api` runner, not the web request. Walk
  away, reload the page, or background the app on your phone, and the turn keeps
  going; when you come back you reconcile against the saved message. A dropped SSE
  socket **resumes via `Last-Event-ID`**: the runner buffers every event and the
  route replays exactly what a reconnecting client missed — **gap-free and
  duplicate-free**.

- **Chat-trail polish.** A hover **copy button** on every reply, a live
  timer-and-token footer while a turn runs, 20 rotating "thinking…" phrases, named
  and timed tool steps, and **stick-to-bottom autoscroll** with a *jump to latest*
  button so reading back never fights the stream.

## Notes

- **One contract, two clients.** The stream is a versioned, client-agnostic JSON
  event contract (`TurnEvent` in `@mantle/client-types`) carried over a Postgres
  `LISTEN/NOTIFY` bus (`@mantle/turn-stream`), bearer-authed from day one. The web
  React client and the **Flutter companion** are both just subscribers — the
  companion's consumer is built and unit-tested against the same
  `GET /api/assistant/turn/:id/stream` endpoint. Design + rationale (the
  durability-vs-liveness split — the journal never carries tokens) in
  [`live-turn-streaming.md`](../live-turn-streaming.md).
- **The responder writes standard Markdown** (`chat_writing`); the rich Pages
  dialect (`rich_writing`) is now Pages-only, so streamed replies render portably
  token by token without a custom parser racing the stream.
- **Schema:** migration **0106** adds the `narrator` AI-worker kind; migration
  **0107** adds `turn_stream_buffer` (the `Last-Event-ID` replay store, short-TTL,
  swept on turn-start). Both ride on top of 0105's durable turn state from 0.66.0.
- **Deploy gotchas:** ⚠️ **detach `rich_writing` from each existing persona
  manually** — the manifest reconcile won't do it. The narrator is a required
  baseline worker, so it auto-seeds on the next upgrade. The label/token changes
  take effect after an **`apps/api` runner restart**.
