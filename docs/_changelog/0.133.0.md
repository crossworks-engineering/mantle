# v0.133.0 — Retrieval that finds the needle, skips the noise, and knows the map

Three retrieval-quality upgrades, all born from a production recall audit.

## Features

- **Exact terms are now findable in passages.** `search_chunks` was purely
  semantic, so a rare literal token — an error code, a field name, a coined
  term — could sit verbatim in an indexed passage and still come back "not
  found" when it embedded poorly. Passage search is now **hybrid**: the
  semantic ranking stays in charge, a keyword arm boosts exact matches, and a
  rescue floor guarantees the top keyword hits a seat in the result even when
  the semantic pool is crowded. The assistant's automatic per-turn passage
  retrieval gets the same upgrade for free.

- **Spreadsheets stop flooding the index.** A flattened workbook used to be
  embedded row by row — thousands of numeric grid passages that crowd real
  content out of search (they were 74% of one production brain's passage
  index). A spreadsheet now indexes as one **profile per sheet**: sheet name,
  column headers, a small sample of rows, and an honest "N of M rows shown —
  full grid via file_read" note. The complete text still lives on the file
  node; nothing is lost, it just stops polluting retrieval. Existing files
  pick this up on their next re-extract.

- **Versioned exports defer to the newest copy.** Weekly dumps and
  `_version_NN` workbooks are recognised as one family (date/version tokens
  in the title); older versions are gently down-ranked in search so retrieval
  prefers the current copy. A nudge, not a hide — every version stays
  readable and findable by title. Tunable via
  `MANTLE_SUPERSEDED_FILE_SALIENCE` (set 1 to disable).

- **The assistant now carries a map of your corpus.** Every responder turn
  includes a compact, cached index of what the brain holds — pages, tables,
  files, notes, tasks, apps, grouped by branch with one-line summaries for
  pages and tables. The assistant stops rediscovering your content one search
  at a time, can answer "what do we have on X" from the map, and knows when
  something *doesn't* exist. Per-agent knob: `memory_config.corpus_map_limit`
  (default 300 entries; 0 disables).

## Under the hood

- Migration 0119 (additive): generated `tsvector` + GIN over
  `content_chunks.text`; hybrid ranking is weighted RRF (0.7/0.3, k=60)
  mirroring node search, with the vector arm still salience-adjusted and
  HNSW-pooled. No-query callers (federation's embedding-only wire format)
  keep the original vector-only path byte-for-byte.
- The corpus map renders branch-grouped and title-sorted so its bytes are
  stable across turns, and rides its **own prompt-cache breakpoint** after
  persona/digests — it re-bills only when corpus content actually changes.
  Snapshot + `load_context` trace output record the map size per turn.
- Sheet-profile chunking triggers on spreadsheet extensions or the flattened
  `# Sheet:` marker; bare CSVs profile as a single sheet. Family grouping
  replaces runs of ≥2 digits — single-digit distinctions ("unit-1" vs
  "unit-2") never group.
