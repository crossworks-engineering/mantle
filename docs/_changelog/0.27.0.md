# 0.27.0 — 2026-06-21

Location awareness, seeded API integrations, and a Tailscale-powered split of the
UI from the core.

## Highlights

- **Location awareness.** Mantle can now use your device's location. The companion
  app attaches a GPS reading to each message; it rides the per-turn context so the
  assistant knows where you are. Ask "where am I?" or "how far am I from a coffee
  place?" and it resolves an address, finds nearby places, and reasons about
  distance + timing. A new **`location`** node type caches resolved places (so the
  same spot isn't re-geocoded), and a `location_awareness` skill teaches the loop.
  Geocoding is **Mapbox** — add a `mapbox` key under **Settings → API keys** to turn
  it on (the tools stay dormant until you do).

- **Seeded API integrations (API Console).** New mechanism that ships ready-made
  HTTP tools out of the box — Mapbox reverse-geocode + place search are the first.
  They reference your key as a vault ref (`{{secret:mapbox/default}}`) and lie
  dormant until the key exists. Adding the next provider (e.g. LocationIQ) is one
  manifest entry, or a live Toolsmith authoring run.

- **Custom API keys.** The keys form (**Settings → API keys**) gained a
  **"Custom / other API…"** option, so you can store keys for non-LLM services
  (Mapbox, and anything your API-console tools call) and reference them in tools.

- **Split UI / core over Tailscale (dev).** The bundled Tailscale node can now
  publish the core's Postgres + object store onto your tailnet (`tailscale serve
  --tcp`), so the UI/app layer runs as a separate client — a laptop dev server
  today, an Electron desktop build tomorrow — against the core on the server, with
  **no inbound ports opened**. See `docs/split-ui-core.md` +
  `docs/remote-db-dev.md`.

## Notes

- The macOS **Connect a device** guide now recommends the menu-bar app
  (`brew install --cask tailscale`) and warns off the CLI-only formula (which
  doesn't start the background daemon).
- New helper scripts for the remote-DB-over-Tailscale workflow: `pnpm db:tunnel`
  (SSH fallback) and `pnpm tailscale:serve` (publish the core on the tailnet).
- `docs/tailscale.md` now documents both directions — outbound (reach inference)
  and inbound (serve the core).
