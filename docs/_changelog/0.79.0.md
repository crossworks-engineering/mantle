# 0.79.0 — 2026-06-27

**Default-behaviour changes now reach existing brains — including removals.**
When a Mantle release changes which built-in skills the assistant carries, that
change now propagates to already-provisioned brains on upgrade. Previously the
boot reconcile could only *add* — so retiring a default skill, or moving it off
the assistant, needed a manual database edit on every box.

## Highlights

- **Skill links converge on upgrade.** The boot reconcile now reconciles each
  agent's manifest-owned skills to the template: it attaches newly-shipped
  defaults *and* detaches a default that was removed — while never touching skills
  an operator authored themselves. It reaches the active responder **by role**, so
  an operator/renamed persona gets the change too, not just the stock `assistant`.

- **The `chat_writing` / `rich_writing` split now self-heals.** The recent change
  that confines the rich Pages dialect to page documents (standard Markdown in
  chat) now lands on existing brains automatically — no manual detach. It also
  shows in **Settings → Config** as an adoptable difference, so you can apply it
  on demand.

- **Operator-authored config stays untouched.** Only product-owned (manifest)
  skills converge; skills, tool-groups, and agents you added yourself are never
  removed. Tool-group links and whole-row deletions remain add-only (disable to
  opt out).
