# 0.31.0 — 2026-06-22

Apps — build small interactive tools by describing them, and run them inside Mantle.

## Highlights

- **The Apps mini-app builder.** A new `/apps` surface — "Pages, but for little
  programs." Ask the assistant (*"build me an app that shows today's weather"*) or
  use the in-surface Assist panel, and the new **Appsmith** specialist writes a
  real React/TypeScript app using Mantle's own UI components, bundles it, fixes
  its own compile errors, and leaves you a live preview. Apps run in a sealed
  sandbox, themed to match the rest of the app (including dark mode and your
  colour theme). See [Apps](../guide/03-using/06-apps.md).

- **Apps reuse your tools and your theme — nothing new to wire.** When an app
  needs live data it calls one of *your* API tools (the same ones the Toolsmith
  builds in the API Console) through Mantle, so API keys stay on the server and
  the app never sees them. Appsmith delegates new integrations to the Toolsmith
  automatically. Each app also gets its own private storage, isolated from every
  other app.

- **Draft → review → publish, like Pages and Tables.** Appsmith's changes land in
  a draft you review in the live preview; **Publish** goes live (only once the app
  builds cleanly), **Discard** reverts. A broken change can never reach the
  published app.

- **Existing brains get Appsmith on upgrade.** The once-per-version boot reconcile
  now provisions any *new* specialist agent the manifest ships (here, Appsmith)
  and wires the assistant's delegation to it — so the feature reaches an
  already-running box on the next version bump, not just fresh installs. It's
  create-only: an agent you've disabled or customised is never touched.

## Notes

- **Schema migration in this release** (`0096`–`0098`): a new `app` node type, the
  `apps` table, and the per-app SQLite registry. Back up before migrating, then
  `pnpm db:migrate` — the enum addition isn't reversible.
- Per-app SQLite databases are stored under `APP_DB_DIR` (defaults to a local
  `.app-dbs/` dir); point it at a persistent volume on a self-hosted box so app
  data survives container restarts.
- Apps render against the components already compiled into your theme; very
  unusual Tailwind utilities not used elsewhere in the app may not style inside an
  app preview.
