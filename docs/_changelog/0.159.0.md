# 0.159.0 — 2026-07-22

Dependency modernisation: **86 outdated packages down to 5**. No feature work —
this release is entirely about getting the tree current, on Next 16, and off a
pg-boss version that upstream had stranded.

> ## ⚠️ This release does NOT deploy cleanly on its own
>
> It contains **pg-boss 12**, and pg-boss's own migration chain is broken between
> v10 and v11 — there is no supported path off our current schema. **Every box
> needs a one-time manual `pgboss` schema rebuild**, or its workers will not
> start. Follow [`docs/pg-boss-12-upgrade.md`](../pg-boss-12-upgrade.md) per box,
> dev brain first. Rolling the image back does not roll the schema back.
>
> After that one hop, pg-boss self-migrates on boot as normal — verified across
> the whole v12 line.

## Highlights

- **Next.js 15 → 16.** Turbopack is now the default for `next build`. Most of the
  v16 breaking changes didn't reach us — `params`/`searchParams` were already
  async, and we have no parallel routes, server actions, AMP, or runtime config.
  `turbopack.root` is now pinned, which fixes Next inferring the *integrator*
  checkout as the workspace root when building from a git worktree.
- **React 19.2.7, Radix, TipTap 3.28, lucide-react 1.x, recharts 3.**
- **drizzle-orm 0.38 → 0.45** across 238 files with zero code changes. Emitted SQL
  was diffed byte-for-byte across versions, and all 136 migration hashes verified
  identical — nothing re-runs on deploy.
- **zod 3 → 4** across 141 files; one breaking change (`z.record()` needs an
  explicit key type).
- **vitest 2 → 4, eslint 9 → 10.** The eslint bump surfaced 14 real defects, all
  fixed — dead initializers that were suppressing TypeScript's own
  definite-assignment checking, and three `throw`s that discarded the caught
  error's cause chain on the extractor's dead-letter path.
- **pg-boss 10 → 12** (see the warning above).

## Fixes

- **`@napi-rs/canvas` was mismatched against `pdf-to-png-converter`** — the pin
  exists so next.config.ts's webpack externalisation resolves at runtime, and it
  had drifted to a 0.1.x binding against a library built for 1.0.x.
- **Two `pdfjs-dist` copies could break PDF extraction.** pdfjs compares API and
  worker version strings exactly and its worker config is process-global, so
  password-protected PDF extraction failed after any normal PDF was parsed in the
  same worker. Now pinned to one version.
- **PDF text extraction got materially cleaner.** pdf-parse 2 drops the
  fixed-width whitespace padding that made up ~81% of extracted output — that
  padding was going into every PDF chunk's embedding. Existing indexed PDFs keep
  their old text; only newly ingested ones change.
- A pdfjs worker leak on the wrong-password path (the most common outcome of that
  function) — teardown now runs in a `finally`.
- Dropped three deprecated/vestigial packages: `@types/libsodium-wrappers`,
  `@types/bcryptjs`, `eslint-config-next` (plus the dead `next lint` script).

## Notes

- **bcryptjs 2 → 3**: existing `$2a$` password hashes were verified to still
  authenticate under v3. New hashes are `$2b$`. No migration, nobody locked out.
- **`pg` is now pinned** in `pnpm-workspace.yaml`. drizzle-orm peers on `pg`, so
  two `pg` versions produce two peer-resolved drizzle instances with
  incompatible types — which presents as a wave of type errors nowhere near the
  actual cause.
- **TypeScript 7 is deliberately held back.** `typescript-eslint` declares
  `typescript >=4.8.4 <6.1.0`, so adopting it would take the lint gate offline.
  Revisit when that peer range includes 7.x.
- `next build` no longer prints `size` / `First Load JS` (removed in Next 16 as
  inaccurate under RSC).
- Full working notes: [`docs/dependency-upgrade-2026-07.md`](../dependency-upgrade-2026-07.md).
