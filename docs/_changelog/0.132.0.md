# v0.132.0 — Share whole categories with a peer

## Features

- **Category peer-shares.** Sharing with a federation peer no longer has to be
  node-by-node: the peer detail pane at Settings → Peers gains a **"Share whole
  categories"** block — one switch per category (Pages, Notes, Files, Contacts,
  Tables, Events, Tasks) with its item count. A category switch is a **standing
  subscription**: every item of that type is readable by the peer, *including
  ones you create later* — flip "Pages" on once and new pages are shared
  automatically, no follow-up granting. Turn it off and the peer is back to
  exactly what was individually shared; the per-item picker stays for
  cherry-picking and the two coexist.

- **Deliberately bounded.** Which categories can be shared wholesale is a
  server-enforced allowlist. Secrets and peer records are never offered, and
  email + journal are excluded by design — they're the owner's private corpus.
  Individual emails or journal entries can still be shared one at a time.

## Under the hood

- New `peer_share_scopes` table (migration 0118, additive) with the same
  revoke-don't-delete audit shape as per-node grants. Grant resolution stays in
  one choke point: the effective grant is *(id explicitly shared) OR (type
  category-shared)*, resolved at query time as a composable search predicate —
  never materialized into id lists — and applied uniformly across federation
  queries, passage search, and node reads. No wire-protocol change; older
  peers simply see richer results.
