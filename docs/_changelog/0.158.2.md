# v0.158.2 — team-admin gets a Members tab

`/team-admin` opened on a Chats pane previewing a transcript that can
never fill again. The Forum superseded 1:1 Team Chat and the write path is
genuinely closed (`POST /api/team/turn` 410s unless
`TEAM_CHAT_POST_ENABLED=1`), so `team_messages` cannot grow on any brain
provisioned since. The tab was also hosting three surface-wide settings that
have nothing to do with chat.

## Changed

- **Chats → Members**, rebuilt person-first. The detail pane is the member's
  forum activity: their posts paired with the agent answer each drew (with
  trace deep-links), the topics they started, the requests they filed, and
  their access log — what the chat thread used to show, plus requests and
  uploads, which were previously only reachable from a global list.
- **Pre-Forum transcripts survive** as a collapsed "Chat archive", rendered
  only when a member actually has one. History is demoted, never hidden, and
  a brain that never had chat never sees a ghost of it.
- **The unread badge means something again** — their posts the owner hasn't
  read in the topic, from `forum_read_cursors`. It clears by opening the
  topic, not by glancing at the row. The frozen chat cursor advances only for
  members carrying an archive, so `team_chat_list` stays honest on old brains
  without writing on new ones.
- **Ordering follows forum activity**, then membership date, so a freshly
  enabled member is visible without scrolling.
- **New Settings tab** takes private reads, hub app and curated Dashboard tags
  out of the member sidebar — none of them is member-scoped.
- The empty state no longer points at a surface that 410s.

## Added

- Content layer: `listForumMemberActivity`, `listForumPostsByContact`,
  `countForumPostsByContact`, `listForumTopicsByAuthor`, plus a `contactId`
  filter on `listTeamRequests`. The reply lateral requires no intervening
  member post, so in a busy multi-author topic nobody is credited with someone
  else's answer.

## Not covered

These readers are owner-scoped and deliberately unfiltered for visibility (the
owner sees private topics) — flagged in the docs so they are not reused on a
member-facing surface. `docs/team-chat.md` carries a FROZEN banner and records
the one real regression this exposed.
