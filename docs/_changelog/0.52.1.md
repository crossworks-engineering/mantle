# 0.52.1 — 2026-06-24

A stray far-out cell in a spreadsheet no longer explodes an import into thousands
of empty columns.

## Highlights

- **Drop phantom empty columns on spreadsheet import.** Importing a real-world
  workbook produced a degenerate table — one sheet parsed to 16,358 columns
  (mostly empty `Column N`s), ballooning the indexed text to ~920 KB and
  polluting the brain's vector space. Cause: `parseSheet` set the grid width from
  the *widest* row, so a single stray formatted/merged cell parked far out (col
  16k) widened every row, and blank headers became placeholder columns. Fix: keep
  a column only if it carries signal — a non-blank header **or** at least one
  non-blank body cell.

## Notes

- Columns stay in source order; placeholder names keep their **original index**
  so they still map to where they sat in the sheet. A real header with an
  entirely empty body is still kept (an intentional column).
- Empty *rows* were never the problem (already dropped twice via SheetJS
  `blankrows:false` + the `nonEmpty` filter).
- Tests: sheet-to-grid 10/10 (new: phantom far-cell trim + empty-bodied-named
  column kept); files suite 152/152. No schema migration in this release.
