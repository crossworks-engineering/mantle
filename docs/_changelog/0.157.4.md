# v0.157.4 — Runner queues slice 2: worker agents + audits (still dark)

Workers and the audit contract land on the slice-1 spine — the full
plan → worker → audit → redo → respond loop now runs end-to-end, still dark
behind `MANTLE_RUNS`. Details: [docs/runs.md](../runs.md).

## What changed

- **`worker` agent role** (migration 0131): workers are templates — each
  `worker_invoke` item spawns a fresh agent turn from the template. The
  default "Worker agent" is ensured lazily on first use (not in the manifest
  while dark). `model = 'inherit'` (the default) runs steps on the
  responder's model/provider/key at execution time; a duplicate worker on a
  cheaper model is the opt-in cost knob. Per-step routing via
  `worker: '<slug>'`; adding a worker never auto-fans-out.
- **`worker_invoke` execution**: whole agent turn per item, claimed under a
  per-run concurrency cap (default 3, `MANTLE_RUNS_WORKER_CONCURRENCY`;
  completions emit slot-release wake-ups). Depth-2 + empty allowlist makes
  propose-don't-mutate structural (`run_*`/`invoke_agent` refuse). Evidence
  is MECHANICAL — the tool-loop's own ledger on the item result; the full
  reply spills to a `read_result` handle. Cost recorded at whichever model
  actually ran.
- **Audit contract** (plan §7): audits promote into RESUME turns (verdict
  deadline 30 min so nothing wedges); the prompt carries the proposal, the
  recorded ledger, and mechanical pre-check auto-flags. New `run_audit` tool
  records `pass`/`redo` — blocking findings only for redo (anti-nitpick),
  redo supersedes + appends a fresh attempt with findings/directive attached,
  one redo max then `needs_human`.
- **Acceptance metric**: per-worker first-pass acceptance rate on
  `/debug/runs`; worker role selectable in /settings/agents; compiled state
  shows worker routing, proposals, and verdicts.
- Engine tests grow to 20: audit-resume promotion, verdict coherence, the
  full redo cycle (supersede/append/promote/cap → needs_human, counter
  integrity), worker-cap claim gating + slot release, pre-check flags.

## Deploy

- Migration 0131: enum-only (`agent_role` + `'worker'`). No behaviour change
  while `MANTLE_RUNS` is unset.
