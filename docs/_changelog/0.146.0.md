# v0.146.0 — Duplicate an agent

One feature, deliberately shaped. **Duplicate** (next to Delete in the agent
editor) clones any agent's full configuration — system prompt, model,
provider and keys, backup route, parameters, tool groups, skills, delegation
list, memory budgets — into the create form as a new agent.

Why clone instead of overriding an agent in place: a clone is
**operator-authored**, so updates never touch it — it's the stable way to
build your own agents on top of the proven system agents, whose definitions
each release keeps in sync. (An in-place override would be silently reverted
by that same sync.)

Guardrails: a clone lands as role **custom** and **disabled**, so a copied
responder can never accidentally enter priority resolution and answer
Telegram or web chat in the persona's place — promoting it is a deliberate
act. The avatar resets so twins stay tellable-apart, and the slug regenerates
from the name as you type.
