# 0.157.16 — MCP `respond_as_agent`: talk to a responder over MCP (real pipeline, nothing persisted)

An MCP client (Claude Code / Desktop) can now send a message **as if it were
the user talking to one of your responder agents** and get its reply. The turn
runs the REAL responder pipeline — composed persona (identity + skills), real
memory retrieval, and the agent's real granted tools, which **execute**
(including `invoke_agent` delegation) — but writes **nothing** to the agent's
conversation store. It's the "Agent Studio sandbox, but with the real tool
loop". Docs: [connecting-claude.md](../connecting-claude.md) §"Talking to a
responder agent".

## Added

- **`runSimulatedResponderTurn(ownerId, opts)`** (`@mantle/assistant-runtime`,
  `run-sim-turn.ts`) — mirrors the `runAssistantTurn` read path (agent
  resolution, prefs, api key, adapter, pre-loaded retrieval context) and the
  shared assemble → `runResponderLoop` core, then returns the reply + tool
  trail **minus every write**: no `recordTurn`, no
  `updateAssistantMessageOutcome`, no `lastUsedAt` / `usageCount` bump. The
  prompt's conversation history is **caller-held** (`opts.history`), not the
  stored `assistant_messages` window, so a probe never sees or touches the
  agent's real memory of talking to you. Runs inside its own trace (kind
  `manual`, subject the agent, `data.surface='mcp_sim'`); the trace id is
  captured via `currentTrace()` and returned for a `/traces` link.
- **`respond_as_agent` MCP tool** (`@mantle/mcp-core`, `build-server.ts`) —
  registered on both transports (stdio `apps/mcp` + the HTTP route). Params:
  `message` (required), `agent_slug` (default responder when omitted),
  `history` (caller-held, capped at 40 turns × 8000 chars — over-cap is
  rejected with a corrective, not silently truncated), `exclude_tools`,
  `max_iterations` (clamped ≤ 30), `include_tool_calls` (default true; each
  tool's args clipped to ~500 chars). Delegation works because importing
  `@mantle/assistant-runtime` wires `registerAgentInvoker` /
  `registerHeartbeatTools` at module load.

## Notes

- **Side effects are real, only the record is suppressed.** A tool the
  responder calls still runs (a note gets created; a confirm-gated call lands
  on `/pending`, returned as `pending_ids`). What's suppressed is the
  conversation write — the agent has no memory of the exchange afterward.
- MCP-only tool: no system-manifest / tool-group entry (those govern in-app
  agent grants, not the raw MCP surface).

## Tests

- `@mantle/assistant-runtime` — `run-sim-turn.test.ts` (9): no-persistence
  guarantee (`recordTurn` / `updateAssistantMessageOutcome` never called),
  caller-history-not-stored-window, `excludeToolSlugs` reaches the loop,
  `maxIterations` clamp, empty-reply flag, and the missing-agent / missing-key
  error parity with the real turn.
- `@mantle/mcp-core` — `build-server.test.ts` (8): the wrapper's input caps
  (over-cap turns / over-long entry rejected without calling the engine), the
  `include_tool_calls` projection, arg clipping, option forwarding, and engine
  errors surfaced as `isError`.
