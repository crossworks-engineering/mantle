# v0.149.0 — Search palette (⌘K over the whole brain)

The web app finally gets human-facing search: a global command palette over
`GET /api/search` (the hybrid engine the mobile companion already uses),
summoned from anywhere with **⌘K** (Ctrl+K) or the header magnifier.

## What's new

- **⌘K palette** — one dialog for the whole shell. Type ≥2 characters and get
  server-ranked results across every node type (notes, pages, files, mail,
  tasks, events, contacts, journal, telegram, docs…). Results keep the
  engine's relevance order — no client-side regrouping. Enter (or click)
  routes through the `/n/<id>` resolver, which already knows each type's
  surface.
- **Go to** — palette queries that match a nav item name (Settings, Tables,
  Studio…) surface an instant quick-jump group above the results. No fetch.
- **Passages mode** — Tab (or the footer toggle) switches the same query to
  passage-level search (`mode=chunks`): quoted passage text with its
  node · heading above, select to open the owning node. First consumer of
  chunks mode in any UI.
- **Superseded hints** — results whose content has a living successor carry a
  destructive-toned "Superseded" badge; clicking the badge opens the newer
  copy, clicking the row still opens the original (parity with the API's
  `supersededBy` annotation).
- **Shared node-type → icon map** (`components/search/node-type-icons.ts`) —
  keyed off the search API's own type enum, so the compiler flags a missing
  icon when the enum grows. Fills a gap where three components each carried
  partial maps.

## Mechanics

- Debounced 300 ms, abortable, sequence-guarded fetches (the mention-picker
  idiom): a stale response can never paint over a newer query, and old rows
  stay rendered while the next fetch is in flight (no flicker).
- ⌘K is skipped while typing in inputs/editors (same guard as ⌘B/⌘J) — except
  in the palette's own input, where it closes the palette.
- No new dependencies: cmdk + the existing `CommandDialog` wrapper (which
  gained a `shouldFilter` pass-through).

**Deploy: needs v0.148.x's `/api/search` on the server — ships in the same
image, no migration, no compose change.**
