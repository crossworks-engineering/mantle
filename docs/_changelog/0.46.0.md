# 0.46.0 — 2026-06-24

Configure the Azure app from the Settings UI — no more editing `.env` and
restarting.

## Highlights

- **UI-settable Azure app config (DB → env fallback).** `MS_CLIENT_ID` /
  `MS_CLIENT_SECRET` / `MS_TENANT` / redirect URI are now settable from
  `/settings/microsoft` instead of editing `.env` + restarting — the right fit
  for self-serve per-brain deploys. The UI row wins;
  the `MS_*` env vars are the fallback, so existing env-configured deploys keep
  working unchanged.

## Notes

- `microsoft_config` singleton-per-owner table (**migration 0101**); the client
  secret is sealed (AAD = owner id) like `tailscale_config`, masked in the UI.
- The redirect URI is stored explicitly + surfaced in the form to copy into
  Azure; this drops the `NEXT_PUBLIC_APP_URL` dependency (routes use the request
  origin). A "Reset to environment" button restores env-driven config.
- Migrations 0100 + 0101 still pending apply (prod DB — back up first).
