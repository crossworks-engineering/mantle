# v0.157.3 — Runner queues: slice 1 complete (engine live end-to-end, dark)

The rest of slice 1 on v0.157.2's spine — the runner-queue engine now runs
end-to-end behind its flag: plan → dispatch → execute → sweep → resume →
inspect. Everything stays DARK by default (`MANTLE_RUNS` unset); dogfood on
dev first. Full write-up: [docs/runs.md](../runs.md).

## What changed

- **`run_plan` / `run_append` / `run_state` / `run_cancel`** responder tools
  (new `runs` tool group in the manifest — deliberately NOT attached to the
  persona while dogfooding; grant manually). Plan trees validate with teaching
  errors: unknown tools rejected at plan time, `run_*`/`invoke_agent` banned
  inside items (no recursion), depth/size caps.
- **Runs worker** (`worker_runs` container + `pnpm dev` lane): pg-boss
  handlers for the three run queues plus the every-minute sweep — overdue
  items fail with a structured timeout that drives the completion counter;
  stale ready items are re-dispatched (heals crash-between-commit-and-enqueue
  without outbox machinery); lost resumes are re-sent.
- **Item execution through the ONE tool executor** (`dispatchTool`) — same
  path as the inline loop: central arg coercion, a `run_item` trace per
  execution (new `trace_kind`, migration 0130), structured failures, per-item
  usage + cost. Semantic retries per `retry_policy`; side-effecting items
  never auto-retry.
- **Resume turn**: at-most-once via a `resumed_at` CAS (migration 0130); the
  responder wakes with the compiled run state and its reply lands as an
  ordinary outbound conversation turn.
- **Run view**: `/debug/runs` — recent runs, collapsible item tree, states /
  one-line outcomes / cost roll-ups / trace links.
- Engine test battery grows to 14 (sweep timeout→counter, lost-dispatch heal,
  lost-resume heal + exactly-once claim, retry requeue).

## Deploy

- Migration 0130: additive (`run_items.resumed_at` + `trace_kind` value).
- New compose service `worker_runs` — idles healthy with the feature off; set
  `MANTLE_RUNS=1` in the app env to enable a brain.
