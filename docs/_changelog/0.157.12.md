## The acceptance gate — PASSED

The plan-§8 crash-test ran against the workstation stack (isolated scratch
app + DBOS system databases, so no live runner could see the crashed
workflow): `CRASH_TEST_SHAPE=resume` registers a workflow with the resume
turn's exact step ordering — journaled `claim_resume`, journaled
`record_outbound`, then `process.exit(137)` BETWEEN the outbound record and
workflow completion. Recovery (`Recovering 1 workflows from application
version mantle-runner-1`) replayed to completion with **exactly one claim
and exactly one outbound row**. On that basis the handover-§5 resume-loss
gap is claimed **closed**: a crash after `claimResume` no longer loses the
wake-up, and a crash after the report is recorded no longer double-posts.

## Added

- **Worker groups / panels (WP5)**: `agent_groups` (migration 0133; soft
  member refs) + `worker_group_ensure` / `worker_group_list` MCP tools. A
  `worker_invoke` with `group:'<slug>'` (seq-only) macro-expands at
  plan/append time into `par(one attempt per member)` + a PANEL audit — the
  engine only sees shapes it already executes. Panel judging: every
  attempt's fenced proposal + mechanical ledger in the resume prompt;
  `pass` = the directive IS the synthesis (`audited_items` lists the
  panelists); a blocking verdict escalates `needs_human` — panels never
  rerun automatically (`findPanelWorkerItems` / `isPanelAudit` in
  `@mantle/runs`; the audit's F7 re-scope, semantic as recommended).
- **Channel-routed resumes** (`runs.origin_channel`, migration 0134):
  `run_plan` records a telegram surface at creation; the ROOT resume
  delivers the report back to the originating chat as a JOURNALED step (no
  double-send on crash-replay), records the turn under `channel:
  'telegram'`, and gives the responder loop the telegram surface so
  channel-aware tools behave. Unpaired/disabled chat → loud log + web-only
  fallback; the report is never lost.
- **Stop actuator**: the run view (`/debug/runs`) gained a Cancel button
  (AlertDialog-confirmed) backed by `POST /api/debug/runs/:id` — the same
  `cancelRun` as the `run_cancel` tool (CAS from running|paused), live even
  with the flag off. This closes the handover-§5 "Stop not wired" gap at
  the operator level; a chat-side run card can publish to the same route
  later.
