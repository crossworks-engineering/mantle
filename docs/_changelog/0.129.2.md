# v0.129.2 — Remote embedders verified for real

The **Embedder** pill on System vitals no longer shows a neutral "remote"
for cloud providers — the brain holds the route's key, so it now runs a
real 1-token embed through the configured provider + model (the same test
the Embedding settings use) and reports the truth:

- **Green** — an embed just succeeded; hover shows provider · model ·
  latency.
- **Red** — the route is down or mis-keyed (ingest embedding will fail).
- The pill label now carries the scope: **Embedder · remote** or
  **Embedder · local**.

The dashboard polls every 10 s but the probe is a paid API call, so
results are cached — 5 minutes while healthy, 60 s after a failure. The
local-embedder check (free models-list ping) is unchanged.
