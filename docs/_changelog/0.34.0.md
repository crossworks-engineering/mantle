# 0.34.0 — 2026-06-23

The Apps builder gets a cleaner workspace: a dedicated Code tab with a real file tree and syntax highlighting.

## Highlights

- **Builder and Code are now separate tabs.** The cramped three-column app
  detail view (source + preview + chat) is gone. The **Builder** tab is an open
  two-column workspace — live preview beside the Appsmith chat — so describing a
  change has room to breathe. The **Code** tab is where you read the source.

- **A file-tree sidebar.** The Code tab lists files in an indented tree:
  folders first, then files, alphabetical at each level. Nested paths like
  `components/Foo.tsx` nest under their folder, the entry file is marked with a
  dot, and selecting a file opens it in the viewer.

- **Syntax highlighting that tracks your theme.** The source viewer highlights
  `.tsx`/`.ts`, `.js`/`.jsx`, `.css`, `.html`/`.xml`/`.svg`, `.json`, and
  `.md` (with auto-detection as a fallback). Token colours map onto theme
  tokens rather than a fixed scheme, so highlighting follows your active theme
  and light/dark. Built on the existing `lowlight`/highlight.js registry — no
  new dependency. Line numbers stay pinned when scrolling long lines.

## Notes

- No schema migration in this release.
- The code view is read-only for now; the Appsmith chat remains how you edit an app.
