# 0.38.0 — 2026-06-23

Adopt from template — `/settings/config` can now apply a manifest change, per item.

## Highlights

- **The config checker can act.** `/settings/config` was read-only; now each
  drifted item has an **Adopt from template** button that writes the manifest
  version into your brain on demand — the same syncs the next version-bump
  reconcile would make, applied now to the one item you pick. It updates the
  correct row: a skill body, a tool group's membership, a specialist, the
  persona's structure, or a worker.

- **Per-kind semantics match the reconcile contract.** Skills/tool groups
  overwrite to the template. A **specialist merges**: prompt/model/params are
  overwritten, but tool groups and skills you added survive. The **persona** only
  takes structure (default tool groups, delegation, skills) — never its
  operator-owned prompt/model/params. A **worker** can be created or, on an
  explicit click, re-modeled to its route.

- **Adopt all.** One button applies every adoptable change at once (ordered so
  grants resolve), excluding worker model changes — those stay a deliberate
  per-item click — and never deleting operator-added items.

- **Real line diffs for bodies.** Skill instructions and specialist prompts now
  render a proper line-level diff (via the `diff` library) showing exactly what
  adopting would add or remove, instead of dumping both versions side by side.

## Notes

- No schema migration in this release. Every write goes through the same
  `adoptManifestItem` path the boot reconcile uses, so the UI and an upgrade can't
  diverge. Operator-added (`extra`) items are never adoptable — adopt never
  deletes.
