# 0.48.0 — 2026-06-24

Outlook / M365 mailboxes ingest through the very same pipeline as IMAP — no
parallel path.

## Highlights

- **Microsoft M2 — Outlook mail via the email pipeline.**
  `@mantle/microsoft/outlook/mail.ts` implements `@mantle/email`'s
  `EmailProvider` (`listSince` / `fetchFull` / `listRecent` / `listFromSender`)
  over Graph; the existing `syncAccount` does the contact gate, classification,
  dedup, and node + emails + attachments insert unchanged. One ingestion path
  for both IMAP and Outlook.

## Notes

- A companion `email_accounts` row (`provider='microsoft'` + a new `ms_account_id`
  FK, **migration 0103**) satisfies the `emails` FK and links the OAuth token;
  `outlook/manage.ts` creates / toggles it.
- The `microsoft-sync` worker gains a mail scheduler + queue; the `email-sync`
  worker now filters to `provider='imap'` so the companions don't hit its
  IMAP-only `pickProvider`. The cursor is a monotonic `receivedDateTime`
  watermark (`sync_state.graph.mail.since`), mirroring IMAP's UID watermark.
- `/settings/microsoft` gains an "Outlook mail" opt-in toggle per account.
- v1: Inbox-only, same contact gate. Migrations 0100–0103 still pending apply.
