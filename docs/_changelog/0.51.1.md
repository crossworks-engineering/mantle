# 0.51.1 — 2026-06-24

Sub-pages created by an agent now actually nest, and a malformed table no longer crashes the page that holds it.

## Highlights

- **Sub-pages that nest.** `page_create` now accepts `parent_id` (an existing
  page's id), so an agent asked to "create a sub-page under X" produces a real
  child instead of a stray top-level page. The new page nests in the tree;
  omit `parent_id` for a top-level page.

- **Table-row crash repair.** A malformed page draft — a `tableRow` carrying a
  bare `paragraph` instead of cells, from a bad block edit — made the editor
  throw `RangeError: Invalid content for node tableRow` and white-screen the
  page. `repairTableRows` now wraps any stray non-cell row child back into a
  `tableCell`, so the document renders again with the intended columns restored.

## Notes

- The repair runs on the read path (`getPage`), so existing bad pages self-heal
  on the next open — the fix is persisted back by the lazy backfill, no
  migration. It also runs on `saveDraft` / `commitPage`, so a broken doc can
  never be persisted again, from any authoring path.
- No schema migration in this release.
