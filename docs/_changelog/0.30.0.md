# 0.30.0 — 2026-06-22

Sharper document retrieval, fuller spreadsheet ingestion, and files you can rename.

## Highlights

- **Passage-first retrieval.** For a "what does X say about Y" question, the
  assistant now reaches for `search_chunks` first — answering from the exact
  passages instead of loading whole files into context. The tool descriptions
  and the `tool_grounding` skill were rebalanced so the chunk index (which was
  already being built) is actually used at query time, cutting tokens and latency
  and keeping retrieval bounded as a brain's corpus grows.

- **Shipped skills update themselves on upgrade.** The boot reconcile now
  force-syncs the manifest-owned skill *bodies* to canonical (the same way
  tool-group membership already reconciled), so an improved skill — like the new
  retrieval policy — reaches an existing brain on the next version bump, not just
  fresh installs. Operator-authored skills (your own slugs) are never touched —
  but note that hand-edits to a *shipped* skill revert on upgrade; fork it to a
  new slug to keep customizations.

- **Macro-Excel and Visio now ingest.** `.xlsm` / `.xlsb` workbooks route through
  the bounded SheetJS parser (previously they fell through to a title-only skip),
  and `.vsdx` / `.vsd` diagrams go through Tika. Large spreadsheets are now chunked
  up to the 1 MB retrievable-text ceiling (aligned with every other format)
  instead of being capped at 256 KB — so bulky inspection/data workbooks become
  fully searchable.

- **Rename files and folders.** The files system now supports in-place rename for
  both files and folders — in the `/files` UI (folder header + per-file row) and
  as agent tools (`file_rename`, `folder_rename`, `folder_describe`) granted to
  the assistant. Renaming a folder rewrites every descendant's path in one
  cascade, so a folder full of content moves with its name.

## Notes

- No schema migration in this release.
- The retrieval and skill-sync improvements land on an existing brain via the
  once-per-version boot reconcile (`docker compose pull && up -d`).
- Existing `.xlsm` / `.vsdx` files uploaded before this release keep their old
  near-empty extraction until re-processed; new uploads ingest fully.
