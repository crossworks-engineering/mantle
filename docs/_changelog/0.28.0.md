# 0.28.0 — 2026-06-22

Maps & directions in chat, location in the web assistant, and automatic timezone
when you travel — building on the location awareness from 0.27.0.

## Highlights

- **Routes & maps in chat.** Ask "where's the nearest coffee shop — how do I get
  there?" and the assistant finds the place, plots the route on a map, and gives a
  short driving/walking overview (distance + time + the key turns — an overview to
  find a place, not live turn-by-turn). The map renders **inline in both the web
  assistant and the companion app**. Built API-only (no map SDK): a swappable
  **`mapbox_directions`** tool for the route, a `route_map` step that renders the
  Static Images PNG server-side (your Mapbox key never reaches the client), and a
  `navigation` skill that drives the loop. Needs a `mapbox` key
  (**Settings → API keys**); dormant until then.

- **Share location from the web assistant.** A location toggle in the chat
  composer lets the web assistant know where you are (for directions and
  "what's nearby"), the same way the companion app already does. Off by default;
  your browser's own permission prompt gates it. It uses your device's geolocation
  (not IP), so a VPN doesn't move it.

- **Automatic timezone when you travel.** When your location is in a different
  timezone, Mantle updates your profile timezone so the clock, scheduling,
  reminders, and quiet hours stay correct — **always on mobile**, and on the web
  **only when the fix is trustworthy** (an IP/VPN-fallback fix is ignored). It's
  derived deterministically from your coordinates (offline, exact — even no-DST
  zones like Phoenix), tells you when it switches, and offers to switch back. A
  manual **`set_timezone`** tool is available too, and a manual change is respected
  (it won't fight you until you actually travel somewhere new).

- **Activity spinner that doesn't look broken.** A long-running agent turn now
  keeps its spinner animating and shows a live elapsed timer, instead of freezing
  into an amber "may be stalled" state. Busy, with a clock — not broken.

## Fixes & hardening

- **Large Office files** no longer OOM the extractor — Tika's heap cap is raised
  (512m → 1g).
- **Spreadsheet ingest** can't hang on a phantom/oversized sheet range anymore
  (bounded xlsx parsing).
- **Chat resilience:** an empty/truncated model response is retried instead of
  surfacing as an error (OpenRouter transient).
- **Embedder health** tile now mirrors the resolver's local fallback, so a healthy
  local embedder reads as healthy.
- **Fresh installs** are hardened so a first deploy can't ship a broken stack
  (install-path fixes).
