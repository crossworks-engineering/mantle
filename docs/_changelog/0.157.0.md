# v0.157.0 — Delegation: terminal-edge depth exception (appsmith → toolsmith)

Companion to v0.155.0's app-builder fixes, closing the delegation shape that
made chat-initiated app builds ping-pong: responder → appsmith → *"needs a
data tool"* → back to responder → toolsmith → back to appsmith. Appsmith has
always declared `delegate_to: ['toolsmith']`, but the hard `MAX_AGENT_DEPTH=2`
cap vetoed the hop whenever Appsmith itself ran as a delegate — the exact
flow observed in the field on 2026-07-20.

## What changed

- **`MAX_TERMINAL_EDGE_DEPTH = 3`** (`packages/tools/src/invoke-agent-guards.ts`)
  — a child agent may now delegate ONE level deeper, but only when the target
  is a **terminal specialist**: an agent with no delegates of its own, so the
  chain provably ends at depth 3 (a depth-3 child can never extend it). The
  edge must still be operator-declared — `checkDelegationAllowed` runs first,
  so only `delegate_to`-listed pairs ever reach the exception. Today exactly
  one default pair qualifies: **appsmith → toolsmith**.
- `checkAgentDepth(parentDepth, { targetIsTerminal })` — the dispatch guard in
  the `invoke_agent` builtin looks up the target's `memory_config.delegate_to`
  (only when the base cap would refuse; missing/disabled target fails closed)
  and passes terminality in. The refusal message now teaches the recovery
  move ("state what you need in your final answer so your parent can arrange
  it").
- **Defence in depth mirrored in the runtime** (`packages/agent-runtime/src/
  invoke-agent.ts`): a hand-constructed `InvokeAgentInput` past the base cap
  is re-checked against the target's terminality; nothing runs past depth 3
  under any path.
- Guard tests extended: terminal edge allowed at 2→3, non-terminal refused,
  depth 4 never, fail-closed defaults, teaching refusal text.

Behavioural note: if an operator later gives Toolsmith delegates of its own,
the appsmith → toolsmith hop at depth 3 is automatically refused again —
terminality is read live, not baked in.

## Deploy

- Tag-only — no migration, no compose change, no manifest structure change
  (the appsmith `delegate_to` edge already ships).
