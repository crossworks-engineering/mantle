# v0.156.0 — Stop a running turn from its Journey

The activity column already links every action to its Journey page; a
**running** streamed turn there now has a **Stop** button, so a turn can be
halted from wherever you noticed it — not only from the chat composer.

## What's new

- **Traces carry their cancel handle** — `startTrace` stamps the turn's
  stream id into the trace's `data.turn_id` (no schema change). Delegated
  sub-agent traces inherit the ROOT turn's id, so stopping from a
  specialist's journey stops the whole turn.
- **Journey detail** (`/debug/journey/:traceId`): when the trace is
  `running` and carries a `turn_id`, a destructive **Stop** button posts the
  existing `/api/assistant/turn/:id/cancel` route — same server-side abort
  path as the chat Stop button (v0.153.2/3: generation halts, pending tool
  calls are cancelled, the partial reply is kept). Running journeys also
  poll every 3 s now, so the timeline grows live and the status flip is
  visible without a reload; `running` renders amber instead of green.
- Traces without a turn id (background ingest, heartbeats, rows from before
  this release) simply don't show the button — there is nothing registered
  to abort.

No migration; the `turn_id` lives in the existing `data` jsonb.
