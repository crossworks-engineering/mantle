# v0.144.0 — Typing that stays typed, a leaner default install

A robustness release. The page editor no longer eats your first keystrokes
with a spurious "changed elsewhere" reload, the default install stops
carrying a ~3.3GB local-embedder payload it never uses, and the team
workspace gets an explicit way home.

## Pages: the "changed elsewhere" toast storm is gone

- **Typing right after opening a page no longer reverts.** Re-entering a
  just-edited page seeded the editor's draft revision from a stale client
  cache; the first autosave then tripped the concurrency guard (409), toasted
  "This page changed elsewhere — reloading the latest draft", and reloaded
  the draft over your fresh keystrokes. Detail views now always revalidate on
  entry and the editor only mounts from the fresh response. The same
  stale-seed fix applies to the Tables grid.
- **The guard now compares revisions, not content.** Refetches that merely
  echo your own autosaved draft back (the database round-trip re-orders JSON
  keys) could read as a foreign change and remount the editor mid-typing with
  a stale revision — guaranteeing a 409 loop of toasts. The editor now adopts
  server truth only when the draft revision is genuinely ahead of everything
  it has seen. A real second writer (another device, the Pages agent) still
  triggers exactly one reload onto the newest draft.
- **Autosaves are serialized.** The blur flush, the debounce timer, and the
  long-typing flush could overlap; concurrent saves carried the same
  revision, so one of your own writes would 409 against the other. One save
  in flight at a time, one queued — and Commit waits out an in-flight
  autosave before publishing, then publishes exactly the snapshot it marks
  committed.

## Local embedder: strictly opt-in, and honest about hardware

- **The default install no longer touches the local embedder at all** — no
  ~3GB Ollama image pull, no service start, no ~300MB model download. Online
  embedding (configured in onboarding) is the default; the bundled embedder
  exists for air-gapped setups.
- **A first-class, persistent switch.** `scripts/install.sh --local-embedder`
  (or `MANTLE_LOCAL_EMBEDDER=1` on the one-line installer) records the choice
  in `COMPOSE_PROFILES` in `.env`, so every later `docker compose pull/up` —
  the built-in updater included — keeps honouring it. `--no-local-embedder`
  turns it off again and stops the services. The previous one-off
  `--profile local-embedder up` did not survive updates.
- **Documented hardware floor.** The CPU embedder degrades the whole stack on
  a 16 GB / 8-core VPS once several files are ingested at once — measured,
  not theoretical. The docs now say so, and the in-app sanity check explains
  how to enable the profile properly instead of a dead-end hint.

## Team workspace

- **Dashboard nav item.** The member workspace's section nav gains a
  Dashboard entry at the top — an explicit way back to the overview instead
  of knowing to click the wordmark.
