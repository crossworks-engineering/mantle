# v0.157.1 — Delegation audit follow-up: shared terminality predicate

Hardening from the sanity audit of v0.157.0's terminal-edge depth exception.
The audit confirmed the security invariants (no path past depth 3, allowlist
runs first, fail-closed on missing/disabled targets, TOCTOU harmless because
a depth-3 child is stopped by both its own allowlist snapshot and the depth-4
cap) and found one maintainability weakness, fixed here.

## What changed

- **`isTerminalDelegateConfig()`** (`packages/tools/src/invoke-agent-guards.ts`)
  — the "target is a terminal specialist" check was hand-mirrored in the
  `invoke_agent` dispatch guard and the runtime invoker with two logically
  equivalent but differently-written expressions; two copies of a security
  predicate is how gates drift. Both gates now call the one exported, tested
  predicate.
- Tests lock in the malformed-config semantics: a non-array `delegate_to`
  (e.g. the string `'toolsmith'`) counts as **terminal**, which is correct
  because `checkDelegationAllowed` fails closed on the same shape — an agent
  with a malformed config can never actually delegate. The test asserts both
  facts together so they cannot diverge silently.

## Deploy

- Tag-only; no behaviour change.
