# v0.152.0 — Curated tag sections on the team Dashboard

The owner can now feature groups of shared pages on the `/team` overview by
tag — a curated front page for the team hub, on top of the automatic briefing
sections. Plan + audit trail: docs/team-curated-sections-plan.md; reference:
docs/sharing.md §4.

## What's new

- **Dashboard sections panel on `/team-admin`** — the owner picks page TAGS
  (pref `teamHubTags`); candidates are drawn from tags that actually appear on
  team-visible page shares, each with a live shared-page count. Chosen tags
  render as chart-palette pills; a **dead tag** (0 currently-visible pages)
  shows muted with a tooltip instead of silently vanishing, so stale curation
  is visible at a glance.
- **Curated sections on the `/team` overview** — each chosen tag renders a
  section of up to 5 team-visible shared pages carrying it (team **and**
  public mode — anything a member could already open), newest node update
  first, with title + summary. Items open in the **workspace reader**
  (`/team/pages?s=<token>`) rather than ejecting members to a bare `/s/`
  page.
- **Curation can never widen access** — the pref only groups; the share stays
  the single source of truth for visibility. Tag + share a page to feature
  it; unshare or untag to remove it.
- **Excerpt fallback** — when a page's LLM summary is missing (committed
  moments ago, or never indexed), listings fall back to an honest excerpt of
  its published plaintext (`excerptFromDocText`), ellipsised whenever the
  head was cut so a torn sentence is never presented as complete.

Data: `curatedTeamSections` in `@mantle/content/team-hub`, served by
`GET /api/team/curated`; tag picks via `GET/PUT /api/team-admin/dashboard-tags`.

## Fixes (same-day audit follow-ups)

- Per-tag listing skips an unpaged `count(*)` it discarded — one query per
  section instead of two.
- Excerpt truncation honesty (the ellipsis fix above).
- Admin pills adopt the shared `tagColorClass` idiom + the dead-tag
  indicator.

No schema changes, no migration — pref + API + UI only.
