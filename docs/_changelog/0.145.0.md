# v0.145.0 — A faster assistant: does light work itself, links you straight there

The assistant stops round-tripping every page and table request through a
specialist, answers "point me to X" with a link that opens the right editor,
and can mint a shareable public link for anything shareable — not just pages.
All of it is manifest-carried, so existing brains pick the behavior up
automatically on this update.

## Hybrid delegation — light work inline, heavy work delegated

Production traces showed the forced specialist hop taxing light work: turns
with a delegation averaged over 7 minutes against 46 seconds without, a
quarter of Pages delegations were sub-30-second edits, and for tables even
*reads* were specialist-gated — the assistant's own grounding skill told it
to use `table_sql`, a tool it didn't hold.

- **The assistant now holds the light slices directly**: create a page, write
  or replace its draft, fix a single block; read tables in full
  (schema/query/SQL/aggregate — lookups answer in seconds, not minutes); add
  or update single rows ("log this expense"). Everything stays draft-safe —
  the live-overwrite path, multi-block surgery, splits, schema and column
  work, imports, and all deletes remain with the Pages/Ledger specialists.
- **A new routing skill** tells it exactly which is which — and *why* the
  web-facing specialists stay delegated: Researcher and Reader run without
  write tools on purpose, as the prompt-injection firewall for untrusted web
  content.
- **Delegation hand-offs carry ground truth.** `invoke_agent` gains
  `subject_node_ids` (name the exact page/table, no searching for the wrong
  one) and automatically attaches the user's verbatim message to every child
  prompt — an under-packed hand-off can no longer strand the specialist
  guessing at intent.

## Links to everything

- **Every content tool now returns a clickable `url`** — the universal
  `/n/<id>` permalink that opens the item on its own editing surface:
  contacts open the contact editor, tables the grid, secrets their detail
  (new), pages the page. Absolute URLs, so they work from Telegram and the
  companion app too.
- **"Point me to David's contact" is answered with a link**, not directions.
  A new linking section in the writing skill: where-is questions get links,
  created drafts end with their review link, sources are cited as links.
- **Settings deep-link too.** Every settings master-detail accepts
  `?selected=` — agents by slug, AI workers by kind
  (`/settings/ai-workers?selected=summarizer`), users by email, keys by
  service, plus skills, tools, tool-groups, heartbeats, peers, and the
  config checker. Tools that list agents return the ready-made URL.

## Share anything

- **`node_share` / `node_unshare`**: a read-only public (or team-only) link
  for any shareable item — notes, tasks, events, files, apps, tables,
  folders — not just pages. Confirm-gated, idempotent, same `/s/<token>`
  mechanics as page sharing; pages keep their dedicated pair with the
  sub-page cascade.
- The assistant now knows the line between **owner links** (login required —
  never handed to outsiders) and **share links** (minted deliberately, for
  someone else's eyes).
