# v0.131.0 — Retrieval at scale: semantic federation, capacity dial, self-checks

The scaling release. Everything a brain promises about retrieval quality at
corpus scale is now enforced, measured, and visible — the engineering behind
the "Scaling Retrieval Without Degradation" whitepaper.

## Highlights

**Federated search got a brain.** Asking a peer Mantle a question used to be
a title/summary substring match; a paraphrase missed. `peer_query` now runs
the same hybrid vector + full-text ranking local search uses — the answering
brain embeds your query text in *its own* vector space, so nothing new
crosses the wire and older peers transparently benefit. Results remain
strictly inside your grants (the grant set is enforced as a hard allowlist
inside the search itself), and if the peer's embedder is down, ranking
degrades to text matching instead of failing. New alongside it:
**`peer_search_chunks`** — passage-level search inside shared documents, so a
peer can return the relevant *sections* of a granted page or file, not just
its summary. Older peers without the endpoint say so cleanly.

**Brain capacity, on a dial.** The dashboard gains a **Brain capacity** card:
a gauge reading your corpus against the split policy (watch at 10k documents /
50k passage vectors; split at 20k / 100k), with per-axis bars and
watch-threshold ticks. Green means no action; amber names the playbook (run
recall checks); red says it's time to break out a federated brain. The same
numbers are exposed to agents as the `brain_capacity` tool, so scheduled
checks and the UI can never disagree.

**The brain now checks its own memory.** `recall_eval` runs your golden
recall cases (a note tagged `recall-eval-cases`) through the shipped
retrievers, scores recall@k/MRR, keeps every run as a note, and flags drift
against the previous run. Seed the weekly **brain-health heartbeat**
(`ALLOWED_USER_ID=… pnpm -C apps/web seed:brain-health`, optional
`TG_CHAT_ID`) and grant the new `brain-health` tool group to your responder:
it messages you *only* when capacity leaves green or retrieval quality
dropped — a healthy week is silence.

## Fixes

- **Vector search actually uses its indexes now.** The salience/recency
  ranking adjustments made every vector ORDER BY ineligible for pgvector's
  HNSW indexes — at scale, each search and each conversation turn would have
  full-scanned the table (the pre-0057 latency cliff, quietly reintroduced).
  All vector arms (`search_nodes`, `search_chunks`, and the per-turn
  fact/content retrieval) now scan bare-distance through the index with
  `ef_search` sized to the candidate pool, then re-rank — verified
  result-identical on real data. Fixes the silent 40-candidate cap on the
  200-candidate hybrid pool as well.
- Doc drift: chunk size (~2750 chars, not 1500) and `chunk_limit` default
  (8) corrected in `memory.md` / `embeddings.md` / `recall-eval.md`.

## Notes

- Operator steps for the self-checks: run the seed script, then grant
  `brain-health` to the agent at **Settings → Agents**. Golden cases import
  automatically from the repo set on first seed.
- Federation additions are wire-compatible both ways; no peer coordination
  needed. `docs/federation.md` Phase 6 has the details.
- New docs: HNSW query recipe (`embeddings.md`), capacity & split policy
  (`memory.md` §8a), automated eval (`recall-eval.md`).
