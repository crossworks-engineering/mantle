# v0.141.0 — The hardening release: the pre-Pinnacle audit lands

A five-dimension audit ran over the whole monorepo — infrastructure,
performance, code quality, correctness, security — and this release is the
audit landing: every finding fixed, the drift-prone spots restructured, and a
much bigger test net underneath it all. Little is visibly new; a lot is now
harder to break.

## Features

- **Page drafts are conflict-safe.** Editing the same page from two places
  (desktop + phone, or you + the Pages agent) used to be silent
  last-write-wins — the slower save quietly erased the faster one. Drafts now
  carry a revision (the same etag model Tables uses): a stale save is refused,
  the editor tells you the page changed elsewhere and reloads the latest
  draft, and the agent's block edits re-read instead of overwriting your
  keystrokes.
- **Backup locations are validated.** Pointing scheduled backups at a path
  inside the container's ephemeral filesystem used to report success while
  every dump was destroyed on the next redeploy. Such locations (overlay,
  tmpfs/ramfs, symlinks into them) are now rejected when you save the setting
  — and re-checked at every run.
- **Wedged workers restart themselves.** Every worker (and the agent runtime)
  now emits a liveness heartbeat; a new autoheal sidecar restarts any of them
  whose event loop stalls. A hung sync worker no longer sits dead until
  someone notices.
- **Telegram turns are sturdier.** An oversized or unreadable image no longer
  kills the turn — the responder retries text-only, like the web assistant
  always did. A turn that produces no answer now says so instead of going
  silent. Agent iteration/tool-budget limits from `memory_config` are now
  enforced on Telegram (and Team Chat), and Telegram turns record the same
  thought trails + tool stats the web assistant persists.

## Under the hood

- **One responder core.** The Telegram pipeline had grown into a ~1,000-line
  re-implementation of the web assistant's turn loop — every fix to one risked
  drifting the other (two such drift bugs were found and fixed, see above).
  Prompt/context/tool assembly and the traced tool loop are now single shared
  modules (`assemble-turn.ts`, `responder-loop.ts`) used by web, Team Chat,
  and Telegram; each channel keeps only its own delivery + persistence.
  Likewise `extractNode` (the ingest hot path, ~1,266 lines) is now a short
  orchestrator over nine focused stages, and the long-duplicated `str`/
  `strArr` and `slugify` helpers are centralized (the copies had genuinely
  diverged).
- **CI now blocks drift.** The repo gains a real quality gate: Prettier
  formatting (785 files reformatted once, mechanically), an ESLint flat
  config, and both wired into CI — with the cleanup rules ratcheted to
  `error` after burning the 78-warning backlog to zero. The gate caught a
  latent dead test assertion on day one.
- **Entity resolution uses its indexes.** Trigram fuzzy matching now
  prefilters via the GIN index, alias lookups switched to the containment
  form the array GIN actually serves (verified by EXPLAIN — the old form
  scanned), and new indexes cover aliases + current graph edges. The
  dashboard stops counting a huge cache table on every load.
- **Backups got real.** Scheduled backups now snapshot per-app SQLite
  databases beside the `pg_dump` + table workbooks, with rotation and status
  counts; mini-app schema changes apply transactionally (no half-applied
  schema); every long-running container has log rotation and a memory cap;
  `minio`/`mc`/`ollama` images are pinned so an upstream `:latest` can't
  break an update mid-pull.
- **~130 new tests where it hurts most** (2,161 → 2,293): the Telegram turn
  pipeline (previously untested), Microsoft + calendar sync (delta/deletion
  semantics, ICS edge cases, Graph paging/retry — the code paths that decide
  what gets created and deleted in your brain), the draft-revision etag, the
  backup-location guard, and the new shared responder modules.

## Fixes

- The dev split-brain that produced table 500s on one deployment: the
  per-app DB path now anchors to the monorepo root, matching Tables.
- SSRF egress guard on runtime dispatch and secret-stripped env
  (`*_KEY` segment match) for `run_terminal`.
- `worker_push` gains the unhandled-rejection keep-alive backstop the other
  workers had; a transient DB blip can't kill it mid-batch.
