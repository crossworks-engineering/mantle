# v0.139.0 — The whole documentation tree, browsable in the hub

Following v0.138.0's "Share sub-pages", a shared subtree now shows up in the Team
Hub as a **browsable tree** — not one opaque card, and not a flood of them.

## Features

- **Shared subtrees nest in the hub.** When you share a page and its sub-pages,
  the hub payload now carries the whole set, each tagged with its parent. The
  built-in hub still lists only the top page as a card (no flooding), but a
  **hub app** can render the full hierarchy — an index page with its documents
  as an expandable tree, each opening in the in-hub reader. On NATREF, the
  Master Documentation Index now browses its ~70 SOPs / QMPs / DRDs in place.

## Under the hood

- `listTeamHubSections` returns every active team-mode page share tagged with a
  `parentToken` — the token of its nearest team-shared ancestor page (a
  correlated deepest-ancestor lookup, ltree `<@` + `nlevel`), or null at the
  top. The bridge `HubData.sections` type carries it; the built-in hub filters
  to top-level, so nesting is opt-in per surface. Every section stays an
  openable team-mode share.
- Manifest: page sharing (`page-share`) belongs to the Pages specialist (and the
  persona), not the Ledger/Tables specialist — the v0.138.0 grant to Ledger is
  reverted.
