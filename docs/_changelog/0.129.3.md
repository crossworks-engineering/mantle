# v0.129.3 — PRAGMA table_info for app migrations; Chromium pill

## Fixes

- **Generated apps can introspect their own schema.** The app db-broker's
  blanket `PRAGMA` ban rejected `PRAGMA table_info` too — so an app whose
  idempotent column migration opened with it failed on *every* load (hit on
  a pilot deployment's Document Change Log app). We now allow exactly
  `PRAGMA table_info(<table>)` / `table_xinfo(<table>)`, anchored end-to-end
  so nothing can piggyback; every other `PRAGMA` plus `ATTACH`/`DETACH`/
  `VACUUM INTO` stays blocked. The `app_authoring` skill and the
  `app_db_schema_set` tool description now teach the guard, the exception,
  and both safe runtime-migration patterns.

## Dashboard

- The PDF browser-sidecar health pill on **System vitals** is renamed
  **Browser → Chromium** — clearer next to the other engine pills (Tika,
  MinIO, Postgres), and its tooltip already carries the browserless +
  Chromium versions.
