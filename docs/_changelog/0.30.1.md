# 0.30.1 — 2026-06-22

Big documents finish extracting on slow embedders.

## Highlights

- **Bulky files no longer thrash on a CPU embedder.** A large document chunks
  into hundreds of passages, and on a CPU-only box those embed serially for more
  than pg-boss's default 15-minute job window — so the extraction job expired,
  retried, and eventually dead-lettered without ever finishing. The extract queue
  now gives a single extraction **60 minutes** (`MANTLE_EXTRACT_EXPIRE_MIN`,
  env-tunable) to complete in one clean pass. Paired with the `0.30.0` embed-batch
  tuning, a big spreadsheet now indexes start-to-finish instead of looping.

- **The activity view stops false-flagging long runs.** The "abandoned after N
  min" trace reaper now tracks the job-expiry window (+ a buffer) instead of a
  flat 10 minutes, so a legitimately-long extraction shows as *running*, not
  *abandoned* — only a genuinely orphaned trace (crashed mid-run) gets reaped.

## Notes

- No schema migration in this release.
- Defaults suit a normal box; only a very slow CPU-only embedder needs to tune
  `MANTLE_EXTRACT_EXPIRE_MIN` (or the embed knobs from `0.30.0`). See
  `docs/embeddings.md` → "Throughput on a CPU-only box".
