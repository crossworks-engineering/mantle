# v0.128.0 — Pages export to Markdown, Word, or PDF

The **Word** download on a page is now a **Download** menu with three
formats. Pick Markdown to hand the text to another tool, Word for editing, or
PDF for a clean, shareable document — each generated on the fly, nothing
stored.

## Three formats, one menu

- **Markdown** — the page content as `.md` (via the same `docToMarkdown`
  serializer the tools use).
- **Word** — the existing `.docx` render, unchanged.
- **PDF** — a faithful print of the page. It renders the page's *live* HTML
  through headless Chromium, so callouts, code highlighting, math, tables, and
  images all look exactly as they do on screen.

## WYSIWYG

Markdown and PDF export the page content as-is — no page **name** injected as
a heading, matching the public share surface. Most pages already open with
their own title, so nothing is doubled; the page name still names the
downloaded file.

## Under the hood

- `resolveExport` gained a `format` option (`md` / `docx`); the export route
  takes `?format=md|docx|pdf`. Existing `/api/export/<id>` links keep
  returning Word.
- PDF is rendered against a new owner-only `/print/pages/[id]` surface, driven
  over loopback with the caller's session forwarded so embedded images load
  authenticated. The Chromium browser is shared across requests.
- The Docker image now ships a system Chromium; puppeteer's own download is
  skipped in favour of it.
