# 0.100.0 — 2026-06-30

**Catch broken-box problems before they bite, and never miss an update.** A new
owner-only **Sanity check** that surfaces the kind of misconfiguration which
hides silently — a feature that's "up" but quietly dead — and a fix so a freshly
published release shows up in the update banner within minutes instead of hours.

## Highlights

- **System sanity check (Debug → Sanity check).** A read-only, owner-only panel
  that probes *configuration correctness*, not just liveness: does the object
  store actually have its bucket (a missing one fails every app build and upload
  with "The specified bucket does not exist"), is the in-app updater configured
  (`MANTLE_STACK_DIR`) or silently parked, are the required secrets present and
  well-formed, is the files root absolute and writable, is the public URL a real
  origin (not localhost), is the local embedder serving the configured model,
  is the background-job schema present. Each failing check explains the problem
  and shows the exact fix — including a copy-pasteable command — and nothing is
  changed from the page; it only reports.

- **Update banner surfaces new releases quickly.** The "Update available" chip
  could stay hidden for up to 6 hours after a release went out, if the box last
  checked just before the release was published. The release check now caches a
  "no update" result briefly (and a confirmed update for longer), so a new
  release appears within minutes. As before, **Settings → Updates → "Check now"**
  forces an immediate re-check.
