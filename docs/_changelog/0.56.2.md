# 0.56.2 — 2026-06-24

Opening a big table is instant again — large grids now render only what's on
screen, and selecting a table shows a loading cue instead of a dead pause.

## Highlights

- **Virtualized table grid.** The editor rendered *every* row, so a 3,000-row ×
  18-col table mounted ~54k stateful cell components on open and froze the tab for
  seconds. The grid now virtualizes its rows with `@tanstack/react-virtual`: the
  grid root is the scroll container and only the ~30 rows in/near the viewport are
  mounted (two spacer `<tr>`s carry the off-screen height), so the open is instant
  and scrolling renders rows on demand. The scrollbar, client sort, totals footer,
  and draft/commit model are unchanged — they all still see the full row set.

- **Selection loading cue.** Selecting a table is a server round-trip (the grid
  loads SSR), which felt dead for a beat with no feedback. The clicked list item
  now shows a spinner and the detail pane a "Loading table…" overlay (`pendingId`,
  cleared when the new selection lands).

## Notes

- Adds `@tanstack/react-virtual` to `@mantle/web`. Tradeoff: the browser's
  Ctrl/Cmd-F won't find rows that aren't currently mounted — use the in-grid sort
  or the agent's `table_query` / `table_aggregate` to locate rows in big tables.
- Verified against a 3,000-row grid: ~21 rows mounted (vs 3,000), scrolling swaps
  the window correctly, both loading indicators fire, no console errors.
- No schema migration in this release.
