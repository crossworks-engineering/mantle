Fourth build package of the audited slice-3 plan
([docs/runs-slice-3-plan.md](../runs-slice-3-plan.md) §4 WP4 + the
pause-matrix amendments). Table-engine work; migration 0132; the feature
stays dark behind `MANTLE_RUNS`.

## Added

- **Spend accounting** (`runs.spent_micro_usd`, migration 0132): every
  `completeItem` adds the item's cost under the already-held run row lock
  (plan C5 — race-free by the lock-ordering rule; increments only when the
  item CAS actually landed; failed items count, cost honesty).
- **Budget auto-pause**: `run_plan` takes `budget_usd`; crossing the budget
  CASes the run `running → paused` (a new status, 0132's CHECK) and queues a
  `run_budget` "raise or cancel?" pending row on the same approval surface
  as `ask_human`. **Approve** raises the budget by one more original budget
  on top of what was spent, resumes the run, shifts frozen READY
  audit/question deadlines by the paused duration, and re-emits the parked
  work inline; **reject** cancels the run (`applyBudgetDecision` in
  `@mantle/runs`).
- **`item_cap` enforcement** (default 200): `createRun` / `appendChildren`
  count nodes under the run lock and refuse past the cap with a teaching
  error (`ItemCapError`) — the runaway-append backstop.

## The pause matrix (the audit's amendments, implemented)

- Pause gates NEW work only: the refusal lives in `claimItem` /
  `claimWorkerItem` — promotion proceeds, so a paused seq run can never
  wedge on a queued child nothing re-promotes (amendment 1 / finding F2).
- A finished run is never paused: the pause CAS runs AFTER completion
  bubbling, and `finalizeRun` + `cancelRun` CAS from `('running','paused')`
  — crossing on the final completion finishes the run; in-flight
  completions finish a paused run with the one resume; a paused run stays
  cancellable (amendment 2 / finding F3).
- Deadlines: running items keep their clocks even while paused (a hung item
  stays killable); READY audit/ask_human deadlines freeze (sweep duty 1
  skips them on paused runs) and shift by the paused duration on resume;
  resume turns refuse on paused runs BEFORE `claimResume`, so the wake-up
  stays re-sendable (amendments 3+4 / finding F6).
- Sweep: duties 2/2b skip paused runs (re-dispatching refused claims is
  churn); janitor duty 4b expires `run_budget` rows whose run left `paused`
  some other way.
