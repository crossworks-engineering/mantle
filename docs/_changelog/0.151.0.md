# v0.151.0 — Maintenance tab (run `pnpm maintain` tasks from the UI)

Phase 3 of the maintenance-runner plan (docs/maintenance-runner.md), brought
forward so admins without terminal access can run maintenance.

## What's new

- **Maintenance tab on `/debug/integrity`** — lists every registry task
  grouped by kind (recurring hygiene / remedies / ops; retired backfills
  collapsed behind a click), with **Preview** (dry-run) and **Apply/Run**
  buttons. Output streams into a console pane below (polled while the run is
  in flight), with state, exit code, and a Cancel button.
- **Same rails as the CLI, enforced server-side** — a shared pure
  `planRun()` (`lib/maintenance/run-args.ts`) backs both the routes and the
  confirm dialogs: live runs of llm/embedding-cost tasks require an explicit
  spend confirmation, retired backfills require a retired confirmation,
  missing env fails fast, and positional-arg tasks (the backup snapshots)
  stay CLI-only. The UI cannot bypass any of it.
- **Single-flight run store** (`lib/maintenance/run-store.ts`) — spawns the
  task's script exactly as `pnpm maintain` does, line-buffers stdout+stderr
  (capped at 2000 lines), maps exit codes, kills runaway runs after 30 min,
  and supports cancel (SIGTERM). One run at a time, app-wide.
- Routes (owner-gated like all `/api/debug/**`): `GET /api/debug/maintenance`,
  `POST`/`GET /api/debug/maintenance/run`, `POST …/run/cancel`.

No schema changes. No migration. Deploy note: the web container must include
`apps/web/scripts/` + `tsx` (it does — same image the workers run from).
