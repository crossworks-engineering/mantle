# 0.120.1 — 2026-07-07

**Every block in a page is addressable again — and stays that way.** A page
could end up with two (or more) blocks sharing one internal block id: the
editor copied the id when you split a paragraph with Enter, and again when
you copy-pasted a block within a page. Everything still *looked* right, but
the assistant's block-level edit tools address blocks by id — with a
duplicate, only the first twin was reachable, so a targeted edit could
silently land on the wrong paragraph or report a block "not found" that's
plainly on screen.

Fixed at both ends:

- **The editor no longer mints duplicates.** Splitting a block gives the new
  half a fresh id, and pasting a copied block re-mints the copy's id on the
  spot — the document never holds two blocks with one id, even mid-edit.
- **Existing pages heal themselves.** Any doc or draft that already carries
  duplicated ids is repaired automatically the next time it's read or saved
  — the first occurrence keeps its id (so existing references stay valid),
  later ones get fresh ids. No migration, nothing to run: opening the page
  is enough.

Also restores a small contract the block tools promised all along: updating
a block keeps its id afterwards (it had quietly started changing on every
update), so a follow-up edit to the same block never needs a fresh listing.
