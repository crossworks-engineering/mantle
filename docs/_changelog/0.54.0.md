# 0.54.0 — 2026-06-24

Pages and notes export to Word, tables export to Excel — from the surface header
or as an agent tool.

## Highlights

- **Render content nodes to Office documents.** Pages & notes → `.docx` (Word),
  tables → `.xlsx` (Excel). Shared renderers live in `@mantle/content`:
  `renderDocx` walks the same ProseMirror tree as the email renderer; `renderXlsx`
  maps the typed `TableDoc` including currency / percent / checkbox formatting +
  a totals row. Notes (markdown) reach `renderDocx` via `markdownToDoc`, so one
  renderer covers pages + notes; page images are embedded via an injected loader.
  OOXML opens cleanly in Word / Google Sheets / LibreOffice — no separate ODF
  path.

- **Two ways out.** `resolveExport()` dispatches by node type and is used by
  both surfaces:
  - **Web** — a `/api/export/[id]` download route + a **Download** button in the
    page / note / table detail headers.
  - **Agent** — an `export_node` tool that saves under `/files/exports`, gated by
    a new `export` tool group granted to the persona + Pages / Ledger.

## Notes

- `docx` / `exceljs` are added as `@mantle/content` deps and externalized for the
  webpack build (reached through the transpiled workspace package).
- Tests: `render-export.test.ts` covers the docx + xlsx renderers.
- No schema migration in this release.
