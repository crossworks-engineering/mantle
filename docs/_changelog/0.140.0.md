# v0.140.0 — Page and filter the integrity live view

The **Debug → Integrity → Live** view showed only the 40 most-recent nodes
across *all* content types at once, so a brain with lots of files or email
surfaced just a couple of pages — the rest were unreachable. It now filters by
type and pages through the whole corpus.

## Features

- **Type filter, order toggle, and pagination on the live view.** Filter the
  live landed view to a single content type (Notes, Pages, Files, …), flip
  between **newest** and **oldest** first, and page through every matching node
  (25/page) with a running total — instead of a fixed 40-row recency window
  shared across all eight types. New content still appears automatically, and
  the page steps back if a deletion empties it.

## Under the hood

- `listLanded` gains `offset` + `order`; a new `countLanded` returns the total
  for the active type filter. Both share one `landedWhere` SQL fragment so the
  page and its total can never drift. `/api/debug/integrity/landed` accepts
  `offset` and `order` and returns `total` on the report.
- The client drives a `<ListPager>` from `report.total` with local page state;
  changing the filter or order resets to page one.
