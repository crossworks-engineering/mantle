# 0.42.0 — 2026-06-24

Agent approvals now follow you to wherever you last spoke, instead of always
landing in Telegram.

## Highlights

- **Route agent approvals by last channel.** Approvals now follow
  `reminderChannel` (the sticky last-communication-channel signal) rather than
  always going to Telegram:
  - **mobile** → the in-app `/pending` queue + a device push deep-linking to
    `/pending`; the Telegram card is skipped.
  - **telegram / unset** → the one-tap Telegram Approve/Reject card (as before),
    and the mobile push is suppressed to avoid double-notifying.

## Notes

- The in-app badge (`pg_notify`) still fires regardless of channel.
- `pushApproval` gains a `'wrong_channel'` skip reason. Mirrors the existing
  reminder-delivery routing.
- No schema migration in this release.
