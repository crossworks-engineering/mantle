# 0.50.0 — 2026-06-24

Subscribe to any calendar by its iCal URL and its events flow into the brain —
searchable, in the events UI, and in the knowledge graph.

## Highlights

- **Provider-agnostic calendar ingestion pipeline.** A calendar pipeline that
  syncs an external calendar into Mantle's existing event nodes. The first
  provider is **universal ICS** (an iCal feed URL): Google's secret iCal address,
  Outlook published calendars, Apple, CalDAV — no OAuth. The ICS provider parses
  with `ical.js` and expands recurrence into per-occurrence events.
- **Synced events are ordinary event nodes.** `@mantle/content`'s
  `upsertExternalEvent` / `listExternalEventUids` / `deleteExternalEvents` dedup
  by `data.external_uid`, suppress reminders (the source calendar already
  notifies), and re-embed only on a real change.

## Notes

- `@mantle/calendar` defines a `CalendarProvider` interface +
  `syncCalendarAccount` orchestrator (full-set ICS deletes-by-absence; delta
  providers tombstone via status). Google / Microsoft OAuth providers plug into
  the same interface later.
- `calendar_accounts` table (**migration 0104**, sealed `feed_url_enc`) + a
  partial dedup index on `nodes(owner_id, data->>'external_uid')`. A
  `calendar-sync` pg-boss worker runs every 2 min via the `worker_calendar`
  compose service. `/settings/calendar` UI: subscribe by iCal URL, enable
  toggle, unsubscribe + nav.
- **Microsoft Graph calendar (M3) is held in favour of this** — it becomes
  "implement `CalendarProvider` for Graph `/me/calendarView/delta`". Migration
  0104 (+ 0100–0103) pending apply. See
  [`calendar-ingest.md`](../calendar-ingest.md).
