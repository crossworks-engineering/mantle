# 0.97.0 — 2026-06-29

**The brain now reads like a database, not a photocopier.** Answering a question
about a long document used to mean pulling the *whole file* into context — which
overflowed the working store, spilled, and got re-sent every step. The responder
now retrieves the relevant section instead, so it scales to large corpora and
costs a fraction.

## Highlights

- **`read_section` — the missing rung.** Between "a few scattered passages"
  (`search_chunks`) and "the entire file" (`file_read`) there's now a tool that
  reads one *section* in full and in order: pass a node id alone for the document
  outline, or a heading / ordinal range for that section. Complete, in-order
  context at section cost. Added to every brain's memory toolkit on upgrade.

- **`file_read` stops dumping huge files.** A large, already-indexed document now
  returns its opening plus a section outline and a pointer to `read_section`,
  rather than dumping its whole extracted text (which spilled and was re-sent
  each loop). `full: true` forces the complete text when you genuinely need it;
  small files are unchanged.

- **Retrieval guidance reframed.** The grounding skill now teaches the ladder —
  injected passages → `search_chunks` → `read_section` → whole file only for
  short docs — and to keep big spreadsheets out of context (search/section, or
  load them into a Table). `search_chunks` results now carry their ordinal so the
  section read is one step away.
