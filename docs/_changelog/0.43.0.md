# 0.43.0 — 2026-06-24

Mini apps stop shipping their own copy of React — one shared runtime is fetched
once and reused across every app and reload.

## Highlights

- **Shared mini-app runtime.** Mini apps no longer re-bundle
  React/ReactDOM/the UI kit/`@host` into every sandboxed iframe. Those are built
  once into a content-hashed, CORS-enabled, immutable-cached `/app-runtime` and
  resolved at load time via an import map injected into the iframe `srcdoc` — so
  the browser fetches + parses one React across every app.
- **−91.5% per-app bundle.** Measured on the Weather app: 214,625 → 18,241
  bytes.

## Notes

- Isolation is unchanged: the iframe keeps its opaque origin and
  `connect-src 'none'` (exfil still blocked — verified). `script-src` names the
  host origin scoped to `/app-runtime/` explicitly (an opaque origin can't match
  `'self'`).
- `packages/app-build` gains the `build-runtime.ts` generator (singleton React
  via code splitting) + manifest; `pre(dev|build)` hooks regenerate the runtime
  so it can't drift from `kit.ts`; the output is gitignored.
- Theme is intentionally untouched here (still capture-and-inline) — the
  `<link>`/live-theme half ships in [0.44.0](./0.44.0.md).
- No schema migration in this release.
