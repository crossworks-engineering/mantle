# 0.33.0 — 2026-06-23

Appsmith can read the web — and new specialist capabilities now reach existing brains on upgrade.

## Highlights

- **Appsmith can research documentation while it builds.** The mini-app builder
  now holds the `research` tool group, so when it's unsure how a library,
  component, or framework API works it can `web_search` for it and `web_fetch`
  the specific doc by URL — instead of guessing. This is for *reading docs while
  coding*; wiring runtime data is still the Toolsmith's job (Appsmith delegates
  HTTP-tool authoring as before), and a mini app still never calls the web
  directly — only through the brokered `host.tools.call`.

- **`web_fetch` is now part of Web research.** The `research` group gains
  `web_fetch` alongside the Perplexity search tiers, so it's a complete
  "read the live web" capability (search + fetch). The Researcher agent picks
  this up too. The Toolsmith already had both (search via `research`, fetch via
  its toolsmith kit) — unchanged.

- **New specialist capabilities reach existing brains automatically.** The
  once-per-version boot reconcile now unions a specialist's manifest tool groups
  onto an already-provisioned specialist agent — additive, never removing. Until
  now, a new group added to an *existing* specialist (like Appsmith gaining
  `research`) only reached fresh installs; provisioning created missing agents
  but never re-granted existing ones. Now Appsmith's web access lands on the next
  version bump. It's additive and respects opt-outs: a specialist you've disabled
  is left alone.

## Notes

- No schema migration in this release.
- Self-hosted upgrade (`docker compose pull && up -d`) applies the new grants on
  the next boot. To force it sooner, or if boot reconcile is disabled, run
  `ALLOWED_USER_ID=<uuid> pnpm -C apps/web seed:appsmith`. The new
  `/settings/config` checker (0.32.0) will flag the grant as missing until it's
  applied.
