# v0.157.5 — Runner queues: pre-deploy audit fixes + durability ADR (still dark)

The independent implementation audit of slices 1+2
([docs/runs-audit-handover.md](../runs-audit-handover.md)) ran before first
deploy; every confirmed finding is fixed here. The feature stays dark behind
`MANTLE_RUNS`. A new ADR records the durable-execution substrate decision:
[docs/adr-runs-durable-execution.md](../adr-runs-durable-execution.md) —
runs v1 stays on pg-boss + the CAS/counter engine; slice 3 is the DBOS
re-evaluation gate; worker turns may move to the `apps/api` durable-turn
runner at any time.

## Fixed

- **`run_audit` banned as a queue item** (HIGH): slice 2 added the tool
  without adding it to either ban list, so a `tool_call run_audit` item could
  record an audit verdict headlessly — bypassing the fresh-context audit turn
  entirely. The list is now a single exported constant
  (`BANNED_ITEM_TOOLS` in `@mantle/tools`) shared by plan-time and
  execution-time checks, with two guard tests: every `RUN_TOOLS` slug must be
  banned, and every slug in the manifest `runs` group must be banned.
- **Lock-order deadlock** (HIGH, reproduced): completion locked child→parent
  while fail_fast cancellation / `cancelRun` locked parent→subtree — a par
  child failing while its running sibling completed deadlocked in Postgres
  (40P01), leaving the sibling to be swept as a spurious timeout. New rule:
  every multi-lock transaction acquires the RUN row lock first
  (`lockRunRow`, engine.ts). Two new race tests; the fail_fast one
  demonstrably deadlocks against the pre-fix engine and passes now.
- **Resume token burned too early**: `claimResume` CASed `resumed_at` before
  the turn's preconditions (agent, api key, adapter), so a transient failure
  permanently swallowed the run's report. The claim now happens after every
  early-return; permanent precondition failures re-send each sweep (visible
  in logs) instead of silently losing the wake-up.
- **Verdicts no longer vanish silently**: `applyAuditVerdict` now checks the
  completion CAS — if the sweep timed the audit out mid-verdict, the caller
  gets a teaching error instead of "verdict recorded".
- **Acceptance metric honesty**: worker steps are classified against actual
  audit verdicts. Unaudited steps no longer count as accepted (a plan with no
  audits scored 100%), and a `needs_human` escalation is its own bucket, not
  a win. `/debug/runs` shows the new buckets.
- **Delegation guard on all run tools**: `run_state` / `run_cancel` now
  refuse at depth > 1 like the rest — a delegated worker can never cancel
  the run it is a step of.
- **Audit prompt injection hardening**: the worker's proposal is fenced and
  labelled untrusted in the audit resume prompt (marker chars stripped from
  the content), so it cannot impersonate the prompt's framing.
- **Smaller**: slot-release wake-ups follow `MANTLE_RUNS_WORKER_CONCURRENCY`
  instead of a hard-coded 3; `run_append` teaches the seq-only audit rule at
  append time when targeting a par group; `requeueForRetry` no longer stamps
  a dead deadline on a ready row; the tool lane executes its batch
  concurrently (it was serially awaited — concurrency 1); the runs worker
  warns at boot when `MANTLE_RUNS` is off while runs are still `running`
  (they neither execute nor time out — cancel or re-enable).

## Notes

- Engine suite grown 20 → 22 (both lock-ordering races), all green against a
  dev-box Postgres; `pnpm verify` green.
