# 0.102.2 — 2026-06-30

**Mini-apps recover instead of hanging when their data directory is missing.**
A follow-up to the persistent app-data work in 0.102.0: an app now heals a
missing database directory on its own rather than getting stuck on a spinner.

## Highlights

- **No more "stuck on Loading…" for `/apps`.** A mini-app keeps its own small
  SQLite database, whose location is recorded when the app is first opened. If
  that directory later went missing — for example on a box where app data lived
  in ephemeral container storage and a stack update recreated the container — the
  app tried to open a database in a folder that no longer existed, failed, and
  hung forever on its initial load. The database layer now re-creates the folder
  automatically on every open, so the app comes back up (with a fresh, empty
  database) instead of hanging. Combined with 0.102.0's persistent app-data
  volume, which prevents the data loss in the first place, `/apps` is now robust
  on both fronts. No user-facing behavior changes for a healthy box.
