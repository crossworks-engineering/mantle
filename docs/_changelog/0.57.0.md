# 0.57.0 — 2026-06-25

The assistant can now read a web page for you. Hand it a link and it pulls the
page's actual content in as context — summarised, quoted, or distilled to the
facts you asked for.

## Highlights

- **New "Reader" specialist.** A delegate agent whose one job is to open a URL
  and read it. The responder delegates page-reading to it the same way it
  delegates search to the Researcher — but the two are split on purpose:
  Researcher *finds* pages on the open web (Perplexity Sonar), Reader *reads* a
  page you already have. It's granted a focused new `web-read` tool group
  (`web_fetch` only — no search tiers, no brain lookups), runs on
  `claude-sonnet-4.6` (`READER_MODEL` to override), and pages through long
  documents via `offset` before handing back a faithful summary / excerpts.

- **How it reads.** `web_fetch` does a guarded HTTP GET (SSRF-protected,
  25 s / 5 MB caps), routes HTML through Apache Tika to readable text (crude
  regex fallback if Tika is down), and returns JSON/markdown/plain text as-is.
  It's `curl` + text-extraction, fully self-hosted — the page never leaves your
  infra. Trade-offs vs. SaaS readers: no JavaScript rendering (SPAs that ship an
  empty shell read thin) and full-text rather than reader-mode extraction, so the
  agent summarises rather than dumps. See [`reader.md`](../reader.md).

## Notes

- Manifest-native: the agent, its prompt, and the `web-read` group all live in
  the system manifest, so fresh brains get it at onboarding and existing brains
  pick it up on the version-bump boot reconcile. To push it to a live brain now:
  `pnpm -C apps/web seed:reader` **and** `pnpm -C apps/web seed:tool-groups`,
  then restart `apps/agent`.
- No new persona grant — the responder already holds `invoke_agent`; it just
  gains `reader` in its delegation allowlist.
- `web_fetch` was already shipping (the Toolsmith uses it to read API docs); this
  release wraps it in a responder-reachable agent. Closes the "`fetch_url`"
  follow-up noted in [`research.md`](../research.md).
- No schema migration in this release.
